## Cache

- 문토 서비스의 필터 캐시
    - Scheduler에서 10분 마다 모든 조건의 Set을 생성
        - 카테고리, 셀호, 클럽소셜링, 인원, 소개팅제외…
    - 모든 정렬의 ZSet을 미리 생성한다.
        - `최신순`, `시작일순`, `기본순(munto)`
- 필터 API 호출
    - 캐시된 데이터가 있으면 내려준다.
    - 없으면 조건에 맞는 set, 정렬에 맞는 zSet으로 Redis에서 intersectStore를 해서 캐시를 생성
    - Code
        
        ```tsx
         private async getIntersectKeys(
            query: SocialingListQuery,
            userId?: number,
          ): Promise<string[]> {
            const keys = [
              CacheKeys.SOCIALING_FILTER_AVAIL_SET,
              ...(await this.getDefaultFilter(query.type, userId)),
              this.getSortKey(query),
            ];
        
            if (this.sharedFilterService.hasSocialingFilterCondition(query)) {
              const filterSetKey = await this.getSocialingFilterSetKey(query);
              keys.push(...(filterSetKey ? [filterSetKey] : []));
            }
        
            return keys;
          }
          
           private async store(

            await this.cacheService.zinterstore(
              resultKey,
              intersectKeys,
              CacheKeys.TTL_FILTER,
            );
        
            if (query.sort === SocialingFilterSortType.munto) {
              await addRandomZsetScoreOnTopItems(this.cacheService, resultKey, {
                length: 20,
                min: 10,
                max: 20,
              });
            }
          }
        ```
        

### 정렬

- 정렬은 `최신순`, `시작일순`, `기본순(munto)`
- 최신순 정렬
    - 점수는 소셜링 이벤트의 생성 날짜를 기준으로 합니다.
    - 공식:
        
        ```tsx
         score = Math.floor(socialing.createdAt.valueOf() / 1000)
        ```
        
- 시작일순
    - 시작날짜를 기준으로 시작일이 가까운순
    - 공식
        
        ```tsx
        score = Math.floor((basedStartDate.valueOf() - socialing.startDate.valueOf()) / 1000) + calculateScoreBySocialingType(socialing)
        ```
        
- 기본순
    - 점수는 경고 점수, 거부 페널티, 특별 호스트 상태, 대기 페널티, 승인율 등 다양한 요소를 고려한 사용자 정의 알고리즘을 사용하여 계산 ([알고리즘 문서](https://www.notion.so/1c0e2bc7639d806fae84d1506b2b601c?pvs=21))
    - 공식
        
        ```tsx
        score = warningScorePenalty * Weight.HAS_WARNING_SCORE_PENALTY + rejectPenalty * Weight.REJECT_PENALTY + specialHost * Weight.IS_SPECIAL_HOST + waitingPenalty * Weight.WAITING_PENALTY + waitingCapacityPenalty * Weight.CAPACITY_UNDER_WAITING_PENALTY + closedPenalty * Weight.CLOSED_PENALTY + approvalRate * Weight.APPROVAL_RATE + adjustScore
        ```
        
    - code
        
        ```tsx
        private async cacheSortedSocialing(
          socialings: SocialingFilter[],
          needReplace: boolean,
          sortBy: SocialingFilterSortType,
        ): Promise<void> {
          const basedStartDate = getFirstDayOfMonth(
            dayjs().add(1, 'year').format('YYYY-MM'),
          );
          const cacheKey = CacheKeys.SOCIALING_FILTER_SORT_TEMP_ZSET(sortBy);
        
          const calculateScoreBySocialingType = (socialing: SocialingFilter) => {
            if (socialing.price >= this.freeSocialingPrice) {
              return 1;
            }
            return 0;
          };
        
          const calculateScore = (socialing: SocialingFilter, adjustScore = 0) => {
            if (sortBy === SocialingFilterSortType.recent) {
              return Math.floor(socialing.createdAt.valueOf() / 1000);
            } else if (sortBy === SocialingFilterSortType.startDate) {
              let score = Math.floor(
                (basedStartDate.valueOf() - socialing.startDate.valueOf()) / 1000,
              );
              score += calculateScoreBySocialingType(socialing);
              return score;
            } else if (sortBy === SocialingFilterSortType.munto) {
              const warningScorePenalty =
                ScoreCalculator.warningScorePenalty(socialing);
              const rejectPenalty = ScoreCalculator.rejectPenalty(socialing);
              const specialHost = ScoreCalculator.specialHost(socialing);
              const waitingPenalty = ScoreCalculator.waitingPenalty(socialing);
              const waitingCapacityPenalty =
                ScoreCalculator.capacityUnderWaitingPenalty(socialing);
              const closedPenalty = ScoreCalculator.closedPenalty(socialing);
              const approvalRate = ScoreCalculator.approvalRate(socialing);
        
              return (
                warningScorePenalty * Weight.HAS_WARNING_SCORE_PENALTY +
                rejectPenalty * Weight.REJECT_PENALTY +
                specialHost * Weight.IS_SPECIAL_HOST +
                waitingPenalty * Weight.WAITING_PENALTY +
                waitingCapacityPenalty * Weight.CAPACITY_UNDER_WAITING_PENALTY +
                closedPenalty * Weight.CLOSED_PENALTY +
                approvalRate * Weight.APPROVAL_RATE +
                adjustScore
              );
            }
        
            return 0;
          };
        
          const adjustSocialings =
            await this.socialingTopFixedReadRepository.findMany();
          const adjustSocialingsMap = transformScoredMap(adjustSocialings);
        
          await this.refreshCachedSocialingSort(
            cacheKey,
            socialings.map((socialing) => {
              return {
                id: socialing.id,
                score: calculateScore(
                  socialing,
                  adjustSocialingsMap[socialing.id] ?? 0,
                ),
              };
            }),
            needReplace,
          );
        }
        ```

## Fan-out

- 피드를 작성, 삭제 하면 피드를 작성한 유저를 팔로우 하는 유저들의 피드 캐시에 피드 추가.
- 피드에 대한 반응 (댓글, 좋아요)이 발생하면 그 피드의 Score가 올라가게 설계
- 동작은 `log streaming` 서버 처럼 ActionHistory 로그를 Kafka로 받아서 처리.
- Feed
    - CREATE
        - 카테고리, 작성 유저 피드, 취향 에디터 캐시
        - 팔로워들 피드에 캐시
    - DELETE
        - 카테고리, 작성유저 피드에서 제거
    - LIKE
        - 팔로워들 피드에 캐시 (스코어 갱신)
- FeedComment
    - CREATE
        - 카테고리, 작성 유저 피드, 취향 에디터 캐시
        - 팔로워들 피드에 캐시 (스코어 갱신)
- Follow
    - CREATE
        - 팔로잉 하는 사람 캐시 상대 피드 캐시
    - DELETE
        - 언팔하는 사람 캐시에서 상대 피드 제거
- Socialing
    - Create
        - 호스트를 팔로우 하고있는 유저들한테 소셜링 캐시
        - `SOCIALING_BY_USER_FOLLOW_SET`
    - Update
        - ItemStatus.CLOSE로 변경될때
        - `SOCIALING_BY_USER_FOLLOW_SET`에서 제거

## 5. Munto Bird

- 작성자:  @장창이
- 작성일:
- 설명:  클럽 채팅방에 참여중인 멤버 리스트를 관리하는 Munto Bird 서버에 대한 설명입니다.

---

- 클럽 채팅방에 유저의 입장, 퇴장을 관리.
    - 기본 채팅방은 그냥 `group`, 클럽 채팅방은 **`Supergroup`** 으로 되어 있음
    - **`Supergroup`** 기존의 그룹채팅과 채팅방 유저 리스트를 관리하는 방법이 달라서 Munto_bird를 이용해서 직접 관리 하고 있음
        - 나중에 샌드버드를 대신해서 자체 채팅서버를 구현하는 것에 대한 테스트 목적도 있었음
    - Redis를 이용해서 관리 하고 있음
        
        ```tsx
          async joinClubChatRoom(
            clubId: string | number,
            userId: number,
          ): Promise<void> {
            const key = `club:${clubId}:room`;
            await this.cache.sadd(key, userId);
          }
        
          async leaveClubChatRoom(
            clubId: string | number,
            userId: number,
          ): Promise<void> {
            const key = `club:${clubId}:room`;
            await this.cache.srem(key, userId);
          }
        ```
        
- 유저 채팅방 입장
    
    ```tsx
      async handleConnection({ user, clubId }: ClubAuthSocket): Promise<void> {
        const room = `club:${clubId}`;
        this.io.socketsJoin(room);
    
        await this.clubChatService.joinClubChatRoom(clubId, user.id);
    
        const chatMessage: ChatMessage = {
          createdAt: new Date(),
          message: `${user.name}님이 입장했습니다.`,
          user,
        };
    
        console.log(`userId: ${user.id} / clubId: ${clubId} connected`);
    
        this.io.to(room).emit(RECEIVE_MESSAGE_EVENT, chatMessage);
      }
    ```
    
- 유저 채팅방 퇴장
    
    ```tsx
     async handleDisconnect({ user, clubId }: ClubAuthSocket): Promise<void> {
        const room = `club:${clubId}`;
        this.io.socketsLeave(room);
    
        await this.clubChatService.leaveClubChatRoom(clubId, user.id);
    
        const chatMessage: ChatMessage = {
          createdAt: new Date(),
          message: `${user.name}님이 퇴장하였습니다.`,
          user,
        };
    
        console.log(`userId: ${user.id} / clubId: ${clubId} disconnected`);
    
        this.io.to(room).emit(RECEIVE_MESSAGE_EVENT, chatMessage);
      }
    ```
