# **1** Introduction **(개요)**

## 1.1 Purpose (목표)

- 이 문서는 문토 체험단(Trial) 기능의 핵심 기능과 요구사항을 정의한 SRS(Software Requirements Specification) 문서입니다.
- 이 SRS는 체험단 기능의 모바일 앱(iOS/Android), 웹 프론트엔드(소셜링 상세 웹뷰), 백엔드 전반에 대한 요구사항을 다룹니다.
- 이 문서는 개발자, 디자이너, 기획자 및 이해관계자가 시스템의 동작과 구조를 명확히 이해하는 데 목적이 있습니다.
- 이 문서는 기존 문토 소셜링 서비스 위에 새롭게 추가되는 체험단 프로모션 기능에 대한 요구사항을 중점적으로 정의합니다.

## 1.2 Product Scope (범위)

체험단(Trial)은 신규 유저의 첫 유료 모임 참여 진입장벽을 낮추기 위한 프로모션 기능입니다. 호스트가 초대권을 구매하여 자신의 소셜링에 체험단 모집을 열고, 대상 유저가 참여 후 리뷰를 작성하면 결제 금액의 80%를 환급받는 구조입니다.

기존 소셜링 시스템(멤버, 결제, 알고리즘, 캐시) 위에 최소한의 확장으로 구현하며, 기존 로직에 대한 변경을 최소화합니다. 

초대권 구매/관리, 체험단 모집 설정, 대상 유저 노출, 신청/결제, 참여 확인, 리뷰 환급, 백오피스 관리 기능을 포함합니다. 상세한 기능 명세는 7장에서 다룹니다.

## 1.4 Terms and Abbreviations (정의 및 약어)

- **체험단 (Trial)**: 호스트가 초대권을 사용하여 신규 유저에게 할인된 가격으로 모임 참여 기회를 제공하는 프로모션 기능
- **체험단 대상 유저 (Trial-eligible User)**: 체험단 참여 자격을 갖춘 유저. 가입 30일 이내, 유료 모임 참여 0회 조건을 충족해야 한다. 본인인증은 노출 조건이 아니며 체험단 모임 목록 진입 시점과 신청 시점에 확인한다
- **일반 유저 (Regular User)**: 체험단 대상 조건을 충족하지 않는 유저. 동일한 소셜링을 일반 모임으로 이용한다
- **체험단 소셜링 (Trial Socialing)**: 초대권이 배정되어 체험단 모집이 활성화된 소셜링. 체험단 소셜링 목록 및 체험단 상세 화면에서 노출된다. 동일한 소셜링이 일반 소셜링 목록에서는 일반 소셜링으로 노출된다
- **일반 소셜링 (Regular Socialing)**: 체험단 모집이 설정되지 않았거나, 일반 소셜링 목록에서 진입한 소셜링. 기존 소셜링과 동일하게 동작한다
- **초대권 (Invite Ticket)**: 호스트가 유료로 구매하는 재화. 소셜링에 체험단 모집을 설정(배정)하면 잔액에서 차감되고, 모임 종료 후 미소진분은 반환된다. 선착순 초대권과 승인제 초대권이 별도 상품으로 존재하며, 유효기간은 구매일로부터 1년이다.
- **초대권 트랜잭션 (Ticket Transaction)**: 초대권의 모든 수량 변동(구매, 배정, 반환, 만료, 환불)을 기록하는 원장(ledger). 구매 건별 만료일을 추적하며 내역 화면을 제공한다

# 2 Overall Description (전체 설명)

## 2.1 Product Perspective (제품 조망)

체험단은 기존 문토 소셜링 서비스 위에 추가되는 프로모션 레이어입니다. 별도의 서비스나 인프라를 구축하지 않으며, 기존 소셜링의 멤버 관리, 결제(Bootpay), 알림 시스템을 그대로 활용합니다.

**기존 소셜링 시스템과의 관계:**

```
기존 소셜링 시스템 (변경 없음)
├── 소셜링 CRUD
├── SocialingMember (grade: STAFF, MEMBER)
├── Order / Payment (Bootpay)
├── Settlement (정산)
├── 정렬 알고리즘 (munto/recent/startDate)
└── 캐시 (Redis ZSET)

체험단 확장 (추가)
├── SocialingMember.grade에 TRIAL 추가
├── Order.trialProductId 추가 (초대권 구매 식별)
├── 초대권 관리 (TrialTicket + TrialTicketTransaction 원장)
├── 체험단 모집 설정 (TrialRecruitment)
├── 체험단 환급 이력 (TrialRefund)
├── 체험단 소셜링 목록/상세 (웹뷰)
├── 리뷰 기반 80% 부분환불 (신규 로직)
└── 호스트 정산 없음 (체험단 멤버 결제분은 문토 수수료)
```

기존 소셜링의 정원, 최소 인원, 채팅방은 일반 멤버와 체험단 멤버가 공유합니다. 정렬 알고리즘과 캐시 구조에는 영향을 주지 않습니다.

## 2.3 Overall Operation (전체 동작방식)

**호스트 플로우:**

```
초대권 구매 → 모집중인 유료 소셜링에 초대권 배정 (잔액 차감) → 모집 조건 설정
→ 체험단 신청 수신 → [승인제: 24시간 내 승인/거절] → 모임 진행
→ 모임 시작 후 24시간 이내에 참석 여부 선택 → [미선택 시: 암묵적 참여 확정]
→ 모임 종료 → 미소진 초대권 반환(스케줄러로 복원)
```

**유저 플로우:**

```
메인 홈 "웰컴 특가" 앵글 → 체험단 소셜링 목록 → 체험단 소셜링 상세 (웹뷰)
→ 본인인증 → 신청/결제 (원가 전액) → [승인제: 승인 대기]
→ 모임 참여 → 리뷰 작성 (모임 종료 후 7일 이내) + 호스트 참석 확정 (24시간 이내, 미선택 시 자동 확정)
→ 두 조건 동시 충족 시 즉시 80% 부분환불
```

**스케줄러:**

- 초대권 만료 처리: 구매일 + 1년 경과 시 자동 소멸 (TrialTicketTransaction에 EXPIRE 기록)
- 초대권 만료 알림: 만료 30/7/3/1일 전 푸시 발송
- 미소진 초대권 반환: 모임 종료 후 배정된 초대권 중 미소진분을 호스트 잔액에 반환 (TrialTicketTransaction에 RETURN 기록)
- 승인제 자동 취소: 결제 완료 후 24시간 내 미선정 시 자동 취소 및 전액 환불
- 참여 암묵적 확정: 모임 시작 시각 기준 24시간 후 호스트 미선택 멤버를 참여 확정 처리
- 환급 기한 처리: 모임 종료 7일 경과 시 미작성 리뷰의 TrialRefund 상태를 EXPIRED로 변경

## 2.4 Product Functions (제품 주요 기능)

본 프로젝트 산출물의 주요 기능을 간략히 기술한다. 상세한 기능은 7장에서 참조한다.

**초대권 (7.1)**

- 호스트가 체험단 모집을 위해 초대권 패키지를 유료 구매한다
- 선착순 초대권과 승인제 초대권은 별도 상품으로 구분된다
- 유효기간은 구매일로부터 1년이며, 만료 시 자동 소멸한다
- 사용 전 환불 정책이 적용되며, 사용 시작 후에는 환불이 불가하다

**체험단 모집 설정 (7.2)**

- 호스트가 모집중인 유료 소셜링에 초대권을 배정하여 체험단 모집을 설정한다
- 모집 조건(인원/성별/나이/신청방식)을 설정한다
- 신청자가 0명일 때만 모집 설정을 변경할 수 있다

**체험단 모임 노출 (7.3)**

- 메인 홈 "웰컴 특가" 앵글에서 체험단 대상 유저에게 체험단 소셜링 목록을 노출한다
- 체험단 소셜링 목록에서 선택 시 체험단 소셜링 상세 화면(웹뷰)으로 진입한다
- 동일한 소셜링이라도 일반 목록에서 진입하면 일반 소셜링 상세로 표시된다

**체험단 신청/결제 (7.4)**

- 체험단 대상 유저가 체험단 소셜링 상세에서 원가 전액을 결제한다
- 본인인증을 필수로 진행하며, 기존 본인인증 모달을 재사용한다
- 체험단 승인제와 소셜링 승인제는 독립적으로 운영된다

**참여 확인 (7.5)**

- 모임 시작 후 호스트가 24시간 이내에 체험단 멤버의 참석 여부를 선택한다. 참석여부를 선택하지 않고 24시간이 지나면 참석한 것으로 간주한다
- 호스트 참석 확정과 유저 리뷰 작성은 독립적으로 진행되며, 두 조건이 모두 충족되면 환급이 실행된다

**리뷰/환급 (7.6)**

- 체험단 멤버가 모임 종료 후 7일 이내에 리뷰를 작성한다
- 호스트 참석 확정과 리뷰 작성이 모두 충족되면 즉시 결제 금액의 80%를 부분환불한다 (검수 없음)
- 호스트 미참석 선택 또는 리뷰 기한 초과 시 환급 대상에서 제외한다

**멤버 관리 (7.7)**

- 체험단 멤버는 기존 SocialingMember 테이블에서 grade=TRIAL로 관리한다
- 호스트가 전체/일반/체험단 탭으로 멤버를 관리한다
- 승인제 초대권 사용 시 호스트가 체험단 신청자를 승인/거절한다

**알림 (7.8)**

- 초대권 만료 전 30/7/3/1일에 호스트에게 푸시 알림을 발송한다
- 체험단 신청, 승인/거절, 리뷰 작성 독려, 환급 완료 등 주요 이벤트에 알림을 발송한다

**백오피스 (7.9)**

- 관리자가 초대권 판매/환불 내역, 체험단 모임 목록, 체험단 멤버, 환급 내역을 조회한다
- 수동 환불 처리 및 체험단 통계 대시보드를 제공한다

## 2.5 User Classes and Characteristics (사용자 계층과 특징)

**체험단 대상 유저 (Trial-eligible User)**

- **자격 요건**: 가입 30일 이내, 유료 모임 참여 0회 (본인인증은 신청 시점에 확인하며 노출 조건이 아님)
- **주요 기능**: 웰컴 특가 앵글 조회, 체험단 소셜링 신청/결제, 리뷰 작성 및 환급
- **특징**: 첫 유료 모임 경험이 없는 신규 유저로, 체험단을 통해 낮은 진입장벽으로 모임에 참여한다

**일반 유저 (Regular User)**

- **자격 요건**: 체험단 대상 조건을 충족하지 않는 문토 계정 보유자
- **주요 기능**: 일반 소셜링 신청/결제
- **특징**: 동일한 소셜링에 일반 멤버로 참여하며, 체험단 관련 기능은 노출되지 않는다

**호스트 (Host)**

- **자격 요건**: 유료 소셜링을 1회 이상 오픈한 이력이 있는 호스트(오픈 후 취소 포함)
- **주요 기능**: 초대권 구매, 체험단 모집 설정, 체험단 멤버 관리 (승인/거절), 참석 여부 선택
- **특징**: 유료 소셜링에만 체험단을 설정할 수 있으며, 체험단 멤버로부터 정산을 받지 않는다

**관리자 (Administrator)**

- **자격 요건**: 문토 운영팀
- **주요 기능**: 초대권 판매/환불 내역 조회, 체험단 모임 관리, 환급 내역 조회, 수동 환불 처리
- **접근 권한**: 백오피스 시스템

## 2.6 Assumptions and Dependencies (가정과 종속 관계)

- 환급 비율 80%는 문토 수수료 비율(20%)에 연동된다. 향후 수수료 비율이 변경될 경우를 대비하여, 체험단 신청(결제) 시점의 환급 비율을 TrialRefund.refundRate에 스냅샷으로 저장한다. 환급 시에는 결제 시점에 기록된 비율을 적용한다
- 초대권 소진 시 동시 신청에 의한 초과 배정(oversell)을 방지하기 위해, 기존 소셜링 정원 관리와 동일한 조건부 UPDATE 패턴을 사용한다 (`usedCount < allocatedCount`일 때만 increment, 실패 시 신청 거절)
- 미모집 페널티(7.2.5) 정책 확정에 따라 ERD가 변경될 수 있다. 페널티 카운트를 별도 컬럼으로 관리할지, TrialRecruitment 상태 기반으로 산정할지가 달라진다.

- **정산**: 기존 Settlement 로직은 변경하지 않는다. 체험단 멤버(grade=TRIAL) 결제분은 호스트 정산 대상에서 제외한다
- **마이그레이션**: enum 추가 및 필드 추가는 non-breaking change이므로 별도 데이터 마이그레이션이 불필요하다

## 6.4 Logical Database Requirements (데이터베이스 요구사항)

**신규 테이블:**

- **초대권 상품 (TrialProduct)**: 선착순/승인제별 초대권 패키지 상품 정보
- **초대권 (TrialTicket)**: 호스트의 초대권 잔액 스냅샷 (hostId + recruitType당 1행)
- **체험단 모집 설정 (TrialRecruitment)**: 소셜링에 배정된 초대권 수, 모집 조건(인원/성별/나이/신청방식), 소진 수량
- **초대권 트랜잭션 (TrialTicketTransaction)**: 초대권의 모든 수량 변동 원장 (구매, 배정, 반환, 만료, 환불). 구매 건별 만료일(expireAt)을 추적한다
- **체험단 환급 이력 (TrialRefund)**: 체험단 멤버별 환급 상태, 결제 시점 환급 비율(refundRate) 스냅샷, 환급 금액

**기존 테이블 변경:**

- **SocialingMember**: `grade` enum에 `TRIAL` 값 추가. 기존 데이터에 영향 없음 (non-breaking change)
- **Order**: `trialProductId` 컬럼 추가 (nullable)
    - 초대권 구매 주문은 `trialProductId IS NOT NULL`로 식별
    - 체험단 관련 컬럼은 추가하지 않고, `SocialingMember.grade = TRIAL` 조인으로 식별

상세 스키마는 [erd.md](https://www.notion.so/munto/erd.md)에서 Prisma 모델 수준으로 정의한다.

## 6.5 Business Rules (비즈니스 규칙)

**체험단 대상 유저 조건:**

- 가입일로부터 30일 이내
- 유료 모임 참여 0회
- 위 두 가지 조건을 충족하면 웰컴 특가 앵글이 노출된다
- 본인인증은 노출 조건이 아니며, 신청 시점에 미완료 시 본인인증 다이얼로그를 표시한다

**초대권 환불 정책:**

| 조건 | 환불 비율 |
| --- | --- |
| 사용 시작 전 + 결제 5일 이내 | 100% |
| 사용 시작 전 + 결제 5일 이후 | 90% |
| 사용 시작 후 | 환불 불가 |
| 1년 경과 (만료) | 환불 불가 |
- **"사용 시작"의 정의**: 해당 초대권이 배정된 체험단 모집(TrialRecruitment)에 1명이라도 신청한 시점. 1명이라도 신청하면 모집 설정 변경, 모집 취소, 초대권 환불이 모두 불가하다

**체험단 유저 취소/환불 정책:**

| 조건 | 환불 |
| --- | --- |
| 결제 후 30분 이내 취소 | 전액 환불 |
| 승인 대기 중 신청 취소 | 전액 환불 |
| 호스트에 의해 거절 또는 내보내기 | 전액 환불 |
| 참여 확정 모임의 진행일 기준 4일 전까지 | 전액 환불 |
| 참여 확정 모임의 진행일 기준 3일 전부터 | 환불 불가 |
| 모임 미참여 | 환불 불가 |
| 모임 진행일 기준 7일 이내 리뷰 미작성 | 환불 불가 |

| 조건 | 환불 비율 |
| --- | --- |
| 사용 전 + 결제 5일 이내 | 100% |
| 사용 전 + 결제 5일 이후 | 90% |
| 사용 시작 후 | 환불 불가 |
| 1년 경과 (만료) | 환불 불가 |

**환급 조건:**

- 환급 조건은 **호스트 참석 확정 (또는 24시간 암묵적 확정)** + **리뷰 작성 완료**이며, 두 조건이 모두 충족되면 즉시 80% 부분환불이 실행된다
- 두 조건은 독립적으로 진행되며, 나중에 충족되는 시점에 환급이 실행된다
- 체험단 멤버는 참석 확정 여부와 관계 없이 모임 종료 후 7일 이내에 리뷰를 작성할 수 있다
- 환급 비율은 결제 시점에 TrialRefund.refundRate에 스냅샷으로 저장되며, 환급 시 해당 비율을 적용한다
- 부분환불 금액의 원 단위 절사/반올림은 기존 소셜링 환불 로직을 따른다
- 부분환불 실패(Bootpay API 오류) 시 처리는 기존 소셜링 환불 실패 처리 로직을 따른다
- 호스트 미참석 선택 또는 리뷰 기한 초과 시 환급 대상에서 제외한다

**체험단 모집 규칙:**

- 유료 소셜링에만 체험단 모집을 설정할 수 있다
- 얼리버드가 적용된 소셜링에도 체험단 모집을 설정할 수 있다. 단, 체험단 멤버에게는 얼리버드 할인이 적용되지 않는다 (원가 전액 결제)
- 체험단 멤버에게는 어떤 할인 및 쿠폰도 중복 적용하지 않는다
- 모집 시작 후 신청자가 0명일 때만 모집 설정(인원, 방식, 성별, 나이)을 변경할 수 있다
- 체험단 승인제와 소셜링 승인제는 독립적으로 운영된다
- 한 유저는 동시에 하나의 체험단 모임만 참여할 수 있다

**폐강 시 처리:**

- 체험단 유저에게 전액 환불한다
- 호스트에게 배정된 초대권을 전액 반환한다 (잔액 복원)

**정산 규칙:**

- 체험단 멤버(grade=TRIAL) 결제분은 호스트 정산 대상에서 제외한다. 기존 Settlement 쿼리에 `SocialingMember.grade != 'TRIAL'` 조건만 추가하면 되며, 별도 정산서나 스케줄러는 불필요하다
- 체험단 멤버 결제 금액 중 환급되지 않은 20%는 문토 수수료(매출)로 처리한다
- 매출 인식 시점:
    - 환급 대상: 환급 실행 시점에 80% 매출 취소, 20% 매출 확정
    - 환급 미대상 (미참석/리뷰 미작성): 모임 종료 + 7일 경과 시점에 100% 매출 확정

## 7.1 초대권

### 7.1.1 초대권 상품

호스트가 구매할 수 있는 초대권 패키지 상품을 제공한다.

- 선착순 초대권과 승인제 초대권은 별도 재화로 관리한다
- 패키지별 상세 구성(인원, 가격, 포함 유형)은 1.5의 체험단 상품표를 따른다

### 7.1.2 초대권 구매

호스트가 초대권 패키지를 유료로 구매한다.

- 유료 소셜링에만 초대권을 사용할 수 있다
- 기존 Bootpay 결제 플로우를 사용한다
- `Order.trialProductId`에 구매한 상품 ID를 기록한다
- 유료 소셜링을 1회 이상 오픈한 호스트만 구매할 수 있다
- 유료 소셜링을 오픈한 후에 해당 모임을 취소하더라도 구매할 수 있다
- 구매 시 TrialTicketTransaction에 PURCHASE 트랜잭션을 기록하며, 구매 건별 만료일(구매일+1년)을 expireAt에 저장한다

### 7.1.3 초대권 잔여 수량 관리

호스트의 초대권 잔여 수량을 관리한다.

- 초대권은 선착순/승인제별로 잔여 수량을 각각 관리한다 (TrialTicket: hostId + recruitType당 1행)
- 모든 수량 변동은 TrialTicketTransaction에 원장으로 기록한다 (balanceBefore → balanceAfter)
- **구매 (PURCHASE)**: 패키지 구매 시 잔액 증가 (+N)
- **배정 (ALLOCATE)**: 소셜링에 초대권 N장 배정 시 잔액 차감 (-N)
- **반환 (RETURN)**: 모임 종료 후 배정된 초대권 중 미소진분을 잔액에 복원 (+N). 미소진분 = TrialRecruitment.allocatedCount - TrialRecruitment.usedCount
- **만료 (EXPIRE)**: 구매 건별 만료일(expireAt) 경과 시 해당 수량 소멸 (-N)
- **환불 (REFUND)**: 초대권 환불 시 잔액 차감 (-N)
- **배정 취소 (DEALLOCATE)**: 모집 취소 시 잔액 복원 (+N) (신청자 0명일 때만 가능)
- 개별 멤버 소진은 TrialRecruitment.usedCount로 추적하며 별도 트랜잭션을 생성하지 않는다
- 호스트는 자신의 초대권 잔여 수량, 변동 내역(TrialTicketTransaction 목록), 만료 예정 수량을 조회할 수 있다

### 7.1.4 초대권 환불

호스트가 미사용 초대권을 환불 요청한다.

| 조건 | 환불 비율 |
| --- | --- |
| 사용 시작 전 + 결제 5일 이내 | 100% |
| 사용 시작 전 + 결제 5일 이후 | 90% |
| 사용 시작 후 | 환불 불가 |
| 1년 경과 (만료) | 환불 불가 |
- **"사용 시작"의 정의**: 해당 초대권이 배정된 체험단 모집(TrialRecruitment)에 1명이라도 신청한 시점 (6.5 비즈니스 규칙과 동일)
- 1명이라도 신청하면 체험단 모집 수정/삭제가 불가하며, 초대권 환불도 불가하다
- 초대권 구매 시점은 TrialTicketTransaction(PURCHASE)의 createdAt 기준이다
- 환불은 백오피스에서 관리자가 처리한다 (7.9.2 참조)

### 7.1.5 초대권 만료

초대권의 유효기간을 관리한다.

- 유효기간은 구매일로부터 1년이다
- 만료 시 미사용 초대권은 자동 소멸하며 환불이 불가하다
- 만료 전 30일, 7일, 3일, 1일에 호스트에게 푸시 알림을 발송한다 (7.8.1 참조)

## 7.2 체험단 모집 설정

### 7.2.1 모임에 초대권 배정

호스트가 모집중인 유료 소셜링에 초대권을 배정한다.

- 호스트가 보유한 선착순 또는 승인제 초대권 중 사용할 수량을 지정한다
- 배정 시점에 초대권 잔액에서 배정 수량만큼 차감한다 (TrialTicketTransaction ALLOCATE 기록)
- 무료 소셜링에는 초대권을 배정할 수 없다
- 얼리버드가 적용된 소셜링에도 초대권을 배정할 수 있다. 단, 체험단 멤버에게는 얼리버드 할인이 적용되지 않는다 (원가 전액 결제)

### 7.2.2 모집 조건 설정

호스트가 체험단 모집 조건을 설정한다.

- **모집 인원**: 배정한 초대권 수 이하로 설정한다
- **성별**: 무관 또는 성별 비율을 설정한다
- **나이**: 누구나 또는 나이 범위를 설정한다
- **신청 방식**: 배정한 초대권 유형에 따라 자동 결정된다 (선착순 초대권 → 선착순, 승인제 초대권 → 승인제)
- 승인제인 경우 호스트가 체험단 전용 질문을 설정한다 (소셜링 승인제 질문과 별개)

### 7.2.3 모집 설정 변경

- 체험단 모집 시작 후 신청자가 0명일 때만 모집 인원, 신청 방식, 성별, 나이를 수정할 수 있다
- 1명이라도 신청한 이후에는 모집 조건 변경이 불가하다

### 7.2.4 모집 종료

체험단 모집이 종료되는 조건을 정의한다.

- 배정된 초대권이 모두 소진되면 모집을 종료한다
- 호스트가 수동으로 모집을 종료할 수 있다
- 모임 종료 후 미소진 초대권은 호스트에게 반환한다 (7.1.3 참조, TrialTicketTransaction RETURN 기록)

**초대권 소진 규칙:**

- 모임 3일 전부터는 모집을 취소해도 초대권은 소진된다
- 체험단만으로 최소 인원이 충족되어도 모임은 진행되어야 한다
- 진행 확정 후 폐강 시 초대권은 소진된다
- 최소 인원 미달로 폐강 시 초대권은 소진되지 않는다

**폐강 시 처리:**

- 체험단 유저에게 결제 금액 전액을 환불한다
- 호스트에게 배정된 초대권을 전액 반환한다 (잔액 복원, TrialTicketTransaction RETURN 기록)

## 7.3 체험단 모임 노출

### 7.3.1 웰컴 특가 앵글

메인 홈 화면에서 체험단 대상 유저에게 "웰컴 특가" 앵글을 노출한다.

- 배너 바로 아래에 "웰컴 특가" 메인앵글을 표시한다
- 체험단 대상 유저 조건을 충족하는 유저에게만 노출한다
    - 가입일로부터 30일 이내
    - 유료 모임 참여 0회
    - 본인인증은 노출 조건이 아님 — 신청 시점에 확인, 7.4.1 참조
- 대상 유저 여부는 서버에서 판단하여 클라이언트에 전달한다
- 기존 메인앵글 로직 및 캐시 구조에는 영향을 주지 않는다

### 7.3.2 체험단 소셜링 목록

웰컴 특가 앵글에서 진입하는 체험단 소셜링 목록을 제공한다.

- 현재 체험단 모집이 활성화된 소셜링 목록을 조회한다 (`TrialRecruitment.status = OPEN` 기준)
- 체험단 소셜링 목록 API(`GET /api/v1/trial/socialings`)를 통해 제공한다.
- 필터: 날짜, 지역, 카테고리 필터를 제공한다
- 정렬: 소셜링 기본 정렬(munto 알고리즘)을 따른다. 별도 정렬 옵션(최신순, 시작일순 등)은 제공하지 않는다

### 7.3.3 체험단 소셜링 상세

체험단 소셜링 목록에서 선택 시 체험단 소셜링 상세 화면을 표시한다.

- 체험단 소셜링 상세는 **웹뷰**로 구현한다
- 체험단 소셜링 목록에서 진입한 경우에만 체험단 상세로 표시한다 (진입 경로 파라미터로 분기)
- 동일한 소셜링을 일반 소셜링 목록에서 진입하면 일반 소셜링 상세로 표시한다
- 체험단 상세에서는 환급 안내(80% 환급 조건), 체험단 신청 CTA를 표시한다
- 앱바(뒤로가기, 설정 메뉴)는 네이티브에서 처리하며, 웹뷰는 앱바 아래 콘텐츠 영역만 담당한다
- 웹-앱 통신은 기존 브릿지 패턴을 재사용한다 ([bridge-events.md](https://www.notion.so/munto/bridge-events.md) 참조)

**웹뷰 선택 근거:**

|  | 앱 네이티브 (Flutter) | 웹뷰 (munto-frontend) |
| --- | --- | --- |
| 기존 코드 상태 | 기술 부채 심각 (19단 중첩, 643줄 God Object 뷰모델, 3중 switch 상태 등) | 구조 깔끔, 조건부 렌더링 패턴 확립 |
| 체험단 분기 추가 공수 | 3~4주 (리팩토링 필수) | 1~2일 |
| 디자인 버전 | 구버전 | 최신 |
| 배포 속도 | 앱 심사 필요 (1~3일) | 서버 배포 즉시 반영 |
| 기존 로직 영향 | 기존 상세 페이지 코드 수정 필요 | 기존 코드 변경 없음 |

## 7.4 체험단 신청/결제

### 7.4.1 본인인증

체험단 신청 시 본인인증을 필수로 진행한다.

- 본인인증은 앵글 노출 조건이 아니며, 신청 시점에 확인한다
- 본인인증 미완료 유저가 신청을 시도하면 본인인증 다이얼로그를 표시하여 인증을 유도한다
- 기존 본인인증 모달을 재사용한다
- 이미 본인인증이 완료된 유저는 다시 인증하지 않는다

### 7.4.2 신청

체험단 대상 유저가 체험단 소셜링에 신청한다.

- 체험단 승인제/선착순은 초대권 유형에 따라 결정되며, 소셜링의 승인제 설정과 독립적으로 운영한다
- **선착순**: 결제 완료 즉시 체험단 멤버로 확정한다
- **승인제**: 결제를 먼저 진행한 후 호스트에게 승인 요청한다. 거절 또는 24시간 자동 취소 시 전액 환불한다. 승인제 특성상 거절에 의한 환불이 빈번할 수 있으며, 이에 따른 PG 수수료 부담이 발생한다
- `SocialingMember` 레코드를 `grade = TRIAL`로 생성한다
- 한 유저는 동시에 하나의 체험단 모임만 신청할 수 있다. 이미 다른 체험단에 신청(승인 대기 포함)한 유저는 추가 신청이 불가하다
- 체험단 멤버에게는 어떤 할인 및 쿠폰도 적용하지 않는다 (원가 전액 결제)
- 유저 입장에서 신청 플로우는 기존 소셜링 신청과 동일하게 느껴진다

**체험단 유저 취소/환불 정책:**

| 조건 | 환불 |
| --- | --- |
| 결제 후 30분 이내 취소 | 전액 환불 |
| 승인 대기 중 신청 취소 | 전액 환불 |
| 호스트에 의해 거절 또는 내보내기 | 전액 환불 |
| 참여 확정 모임의 진행일 기준 4일 전까지 | 전액 환불 |
| 참여 확정 모임의 진행일 기준 3일 전부터 | 환불 불가 |
| 모임 미참여 | 환불 불가 |
| 모임 진행일 기준 7일 이내 리뷰 미작성 | 환불 불가 |

### 7.4.3 결제

체험단 유저가 소셜링 원가 전액을 결제한다.

- 기존 Bootpay 결제 플로우를 그대로 사용한다
- 결제 금액은 소셜링 원가 전액이다 (할인 적용 없음)
- 체험단 유저 결제 주문은 `SocialingMember.grade = TRIAL` 조인으로 식별한다(Order에 별도 orderType 컬럼을 추가하지 않음)
- 결제 수단은 기존과 동일하다 (카드/네이버페이/카카오페이 등)

### 7.4.4 초대권 소진

체험단 멤버 확정 시 TrialRecruitment.usedCount를 1 증가시킨다.

- **선착순**: 신청(결제 완료) 시점에 1장 소진한다
- **승인제**: 호스트가 승인한 시점에 1장 소진한다
- 배정된 초대권이 모두 소진되면 해당 소셜링의 체험단 모집을 자동 종료한다
- 동시 신청에 의한 초과 배정을 방지하기 위해, 기존 소셜링 정원 관리와 동일한 조건부 UPDATE 패턴을 사용한다 (`usedCount < allocatedCount`일 때만 increment, 영향받은 행이 0이면 신청 거절)

## 7.5 참여 확인

### 7.5.1 호스트 참여 확인 방법

모임 시작 후 호스트가 체험단 멤버의 참석 여부를 선택한다.

- 모임 시작 시각 이후부터 호스트가 체험단 멤버별로 참석/미참석을 선택할 수 있다 (모임 시작 전 선택 불가)
- 호스트가 **참석 확정**하면 환급 조건 중 하나가 충족된다. 리뷰 작성이 이미 완료된 경우 즉시 환급이 실행된다
- 호스트가 **미참석**으로 선택하면 해당 유저는 환급 대상에서 제외된다 (리뷰 작성 여부와 무관)
- 모임 시작 시각 기준 **24시간 이내에 호스트가 선택하지 않으면**, 해당 멤버는 암묵적으로 참여 확정 처리한다 (구매확정 방식)
- 암묵적 확정은 스케줄러에서 처리한다 (2.3 참조)
- 유저 화면에서는 호스트의 참석 확정 여부와 리뷰 작성 완료 여부를 각각 표시하며, 두 조건 충족 상태에 따라 환급 진행 상태를 안내한다

## 7.6 리뷰/환급

### 7.6.1 리뷰 작성

체험단 멤버가 모임 참여 후 리뷰를 작성한다.

- 체험단 멤버는 호스트의 참석 확정 여부와 관계없이 모임 종료 후 리뷰를 작성할 수 있다
- 리뷰 작성 기한은 모임 종료 후 7일 이내이다
- 환급 조건: **호스트 참석 확정 (또는 24시간 암묵적 확정)** + **리뷰 작성 완료**가 동시에 충족되면 즉시 80% 부분환불이 실행된다
- 리뷰를 먼저 작성하고 이후 호스트가 참석 확정하면 그 시점에 환급이 실행된다
- 호스트가 먼저 참석 확정하고 이후 유저가 리뷰를 작성하면 그 시점에 환급이 실행된다

### 7.6.2 즉시 80% 부분환불

호스트 참석 확정과 리뷰 작성이 모두 충족되면 즉시 결제 금액의 80%를 부분환불한다.

- 두 조건 중 나중에 충족되는 시점에 자동으로 부분환불을 실행한다 (관리자 검수 없음)
- 기존 `BootpayService.refund` 패턴을 활용하여 부분환불을 처리한다
- 부분환불 금액 = 결제 금액 × TrialRefund.refundRate (결제 시점에 스냅샷으로 저장된 비율)
- 부분환불 금액의 원 단위 절사/반올림은 기존 소셜링 환불 로직을 따른다
- 환불 수단은 원결제 수단으로 역환불한다
- Bootpay API 호출 실패 시 처리는 기존 소셜링 환불 실패 처리 로직을 따른다
- 환급 완료 시 유저에게 알림을 발송한다 (7.8.5 참조)

### 7.6.3 환급 미대상 처리

환급 조건을 충족하지 못한 체험단 멤버를 처리한다.

- **호스트 미참석 선택**: 호스트가 미참석으로 선택한 멤버는 리뷰 작성 여부와 무관하게 환급 대상에서 제외한다
- **기한 초과**: 모임 종료 후 7일이 경과하면 리뷰 작성이 불가하며, 참석 확정 상태이더라도 환급 대상에서 제외한다
- 환급 미대상인 경우 결제 금액 전액이 문토 수익으로 처리된다
- 운영팀이 필요 시 백오피스에서 수동 환급을 처리할 수 있다 (7.9.5 참조)

## 7.7 멤버 관리

### 7.7.1 체험단 멤버 조회

체험단 멤버를 조회하는 전용 API를 제공한다.

- `SocialingMember` 테이블에서 `grade = TRIAL` 조건으로 필터링한다
- 소셜링별 체험단 멤버 목록, 참석 확정 상태 (확정/미확정/미참석), 환급 상태를 조회할 수 있다

### 7.7.2 호스트 멤버 관리

호스트가 소셜링 멤버를 전체/일반/체험단 탭으로 구분하여 관리한다.

- **전체 탭**: 일반 멤버 + 체험단 멤버 전체 목록
- **일반 탭**: `grade = MEMBER` 또는 `grade = STAFF` 멤버
- **체험단 탭**: `grade = TRIAL` 멤버
- 체험단 탭에서는 참석 확정 상태 (확정/미확정/미참석), 리뷰 작성 여부, 환급 상태를 표시한다

### 7.7.3 승인/거절

승인제 초대권으로 모집 시 호스트가 체험단 신청자를 승인 또는 거절한다.

- 호스트가 신청자의 답변을 확인하고 승인 또는 거절한다
- 승인(선정) 후 취소는 불가하다
- 유저의 결제 완료 시점 기준 24시간 이내에 선정을 완료해야 한다
- 24시간 내 미선정 시 신청이 자동 취소되고 전액 환불된다
- 승인 시 초대권 1장이 소진되며 체험단 멤버로 확정된다 (7.4.4 참조)
- 거절 시 초대권은 소진되지 않으며 유저에게 거절 알림을 발송한다 (7.8.3 참조)
- 소셜링 승인제와 독립적으로 운영한다 (소셜링이 선착순이어도 체험단은 승인제일 수 있다)

## 7.8 알림

모든 알림은 기존 문토 알림 채널 정책(푸시/인앱)을 따른다.

### 7.8.1 초대권 만료 알림

초대권 만료 전 호스트에게 푸시 알림을 발송한다.

- 만료 30일, 7일, 3일, 1일 전에 각각 발송한다
- 스케줄러에서 매일 만료 예정 초대권을 조회하여 발송한다

### 7.8.2 체험단 신청 알림

체험단 멤버가 신청하면 호스트에게 알림을 발송한다.

- 선착순: 신청(결제 완료) 시 호스트에게 알림
- 승인제: 신청 시 호스트에게 승인 요청 알림

### 7.8.3 승인/거절 알림

호스트가 승인/거절하면 체험단 유저에게 알림을 발송한다.

- 승인 시: 유저에게 체험단 참여 확정 알림
- 거절 시: 유저에게 체험단 신청 거절 알림 및 결제 취소(전액 환불) 처리

### 7.8.4 리뷰 작성 독려 알림

모임 종료 후 체험단 멤버에게 리뷰 작성을 독려하는 알림을 발송한다.

- 발송 시점: 모임 종료 당일(D0), D+1, D+3, D+7에 발송한다
- **우선순위: P2**

### 7.8.5 환급 완료 알림

부분환불이 완료되면 유저에게 알림을 발송한다.

- 리뷰 작성 → 80% 부분환불 처리 완료 시 유저에게 환급 완료 알림을 발송한다

### 7.8.6 승인 마감 리마인더 알림

승인제에서 24시간 마감이 다가오면 호스트에게 리마인더 알림을 발송한다.

- 마감 전 일정 시점에 호스트에게 미처리 신청 건수와 함께 리마인더를 발송한다
- **우선순위: P2**

## 7.9 백오피스

### 7.9.1 초대권 판매 내역 조회

관리자가 초대권 판매 내역을 조회한다.

- 호스트별 구매 이력 (패키지 종류, 구매일, 결제 금액)
- 초대권 잔여/소진/만료 현황

### 7.9.2 초대권 환불 처리

관리자가 호스트의 초대권 환불 요청을 처리한다.

- 환불 정책(6.5 참조)에 따라 환불 가능 여부를 판단한다
- 관리자가 환불을 승인하면 Bootpay를 통해 환불을 실행한다

### 7.9.3 체험단 모임 목록 관리 (P2)

관리자가 체험단이 설정된 소셜링 목록을 조회하고 관리한다.

- 체험단 모집 중인 소셜링 목록 조회
- 소셜링별 초대권 배정 수, 신청자 수, 소진 수 확인

### 7.9.4 체험단 멤버 조회 (P2)

관리자가 체험단 멤버를 조회한다.

- 소셜링별 체험단 멤버 목록 (신청 상태, 참여 확인 상태, 리뷰 작성 여부, 환급 상태)

### 7.9.5 환급 내역 조회 / 수동 처리

관리자가 환급 내역을 조회하고 예외 케이스를 수동으로 처리한다.

- 자동 환급 내역 조회 (리뷰 작성 → 80% 부분환불 처리 이력)
- 예외 케이스 수동 환불 처리 (CS 대응)

### 7.9.6 체험단 통계 대시보드 (P2)

관리자가 체험단 운영 현황을 대시보드로 확인한다.

- 초대권 판매/소진/만료 추이
- 체험단 신청/참여/리뷰 작성/환급 전환율
- 체험단 참여 후 재참여(일반 멤버 전환)율
