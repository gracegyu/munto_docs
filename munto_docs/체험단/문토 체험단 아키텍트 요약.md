# 문토 체험단(Trial) — 아키텍트 요약

**목적**: 신규 유저의 첫 유료 모임 진입장벽 완화, 호스트 리뷰·신규 유저 확보, 초대권 판매 수익.  
**원칙**: 기존 소셜링(멤버·결제·정산·목록 알고리즘·Redis 캐시)을 **최소 변경**으로 두고, 체험단은 **별도 레이어·별도 정산 타이밍**으로 얹는다.

---

## 1. 비즈니스 한 줄

- **유저**: 원가 **전액** 결제 → 모임 참여(호스트 **참여 확인**) → 모임 종료 후 **7일 이내** 리뷰 → **즉시 80% 부분환불**(검수 없음). 나머지 20%는 문토 수수료.
- **호스트**: **초대권 패키지** 유료 구매 → 모집 중 **유료** 소셜링에 배정·모집 설정 → 체험단 멤버 결제분은 **호스트 정산 없음**(문토 수수료 처리).
- **대상 유저**: 가입 **30일 이내**, **유료 모임 0회**, **본인인증 완료**(웰컴 특가 노출·신청 조건).

---

## 2. 시스템 구성(배포 단위)

| 계층 | 역할 |
| --- | --- |
| **munto-mobile** (Flutter) | 홈 **웰컴 특가** 앵글, 체험단 목록, 초대권 구매, 결제(Bootpay), 리뷰, 참여 확인(QR 등은 앱 측) |
| **munto-frontend** (Next.js) | **체험단 진입 경로**에서만 소셜링 **상세 웹뷰**(일반 목록 진입 시 기존과 동일 상세) |
| **munto-backend** (NestJS) | Trial API, 초대권·모집·환급·스케줄러 |
| **PostgreSQL** | Order, SocialingMember 확장 + Trial 전용 테이블 |
| **Redis** | 기존 필터/정렬 캐시 — 체험단은 **알고리즘·ZSET 구조 변경 없음** |
| **Bootpay** | 체험단 결제·부분환불(기존 `refund` 패턴 재사용) |

**브릿지**: 웹뷰 상세 ↔ 앱은 기존 브릿지 이벤트 패턴 재사용.

## 3. 도메인·데이터(핵심만)

**신규(개념)**

- `TrialProduct` — 선착순/승인제 등 패키지 상품
- `TrialTicket` — 호스트 보유 잔여·만료·환불 상태
- `TrialRecruitment` — 소셜링별 배정 수·모집 조건(인원/성별/나이/신청 방식)

**기존 확장**

- `SocialingMember.grade`: **`TRIAL` 추가**(STAFF/MEMBER와 동일 테이블, 체험단 식별)
- `Order`: **`trialProductId` nullable** — 초대권 구매 주문 식별(SRS 기준). 체험단 참가 주문은 `grade=TRIAL` 등으로 연계.

**정원·채팅**

- 일반+체험단 **정원 합산**, 채팅방 공유(SRS).

## 4. 정책이 아키텍처에 미치는 점

| 주제 | 설계 포인트 |
| --- | --- |
| **승인제** | 소셜링 승인제와 **완전 독립**. 체험단 전용 질문·24h 미선정 자동 취소·전액 환불. |
| **얼리버드** | 동일 소셜링에 **체험단과 동시 적용 불가**(SRS 6.5, 기획 확인서 확정). *(SRS 7.2.1 문장과 교차 검토 필요 — 구현 전 정책 단일화 권장.)* |
| **할인/쿠폰** | 기획 확인서 기준 **체험단은 타 할인·쿠폰과 중복 불가**. |
| **초대권** | 구매일 기준 **1년** 만료·스케줄러; 배정은 소진 아님, **신청/승인 시** 소진; 모임 종료 후 미소진 **반환**. |
| **호스트 초대권 환불** | 미사용 위주, 기간·사용 시작 여부에 따른 단계 환불율; **신청 1명이라도 있으면** 모집 수정·초대권 환불 불가. |
| **유저 환급** | 참여 미확인·7일 초과 → 환급 없음(전액 플랫폼 수익 처리). |
| **정산** | 일반 멤버: 기존 Settlement 유지. 체험단: **모임 종료+7일 후** 별도 배치(리뷰·환급 확정 이후와 충돌 방지). Jira에도 “체험단 정산 별도 스케줄러” 명시. |

---

## 5. 소셜링 알고리즘·목록과의 관계

- 체험단 전용 목록 API는 **`grade = TRIAL` 기준** 등으로 제공.
- **정렬**: 체험단 목록은 **기본순(munto 알고리즘)** 만(SRS 7.3.2 — 최신/시작일 옵션 없음).
- **기존 기본순(munto)** 은 잠재 수익·인기도·판매율·진행확정 가능성·승인율·각종 패널티 등 가중 합산 점수(별도 문서). 체험단은 이 **캐시/수식을 건드리지 않고**, 노출 대상 집합만 체험단 조건에 맞게 조회한다는 전제.
- 체험단 멤버는 **멤버 수 계산에 포함**(하위 호환·정원 논리).
