## **체험단(Trial) 요약**

### **비즈니스 모델**

- 신규 유저(가입 30일 이내, 유료 모임 0회)에게 **첫 모임 참여 진입장벽을 낮추는** 프로모션 기능
- 유저가 **원가 전액 결제** → 모임 참여 + 리뷰 작성 → **80% 환급** → 문토 수수료 20%

### **호스트 측**

- **초대권 패키지 유료 구매** (스타터 10명 ~ 프로맥스 100명, 선착순/승인제 별도 상품)
- 유효기간 1년, 여러 모임에 나눠서 사용, 유료 모임에만 적용
- 모집 중인 유료 소셜링에 초대권 배정 → 모집 조건(인원/성별/나이/신청방식) 설정
- 체험단 멤버로부터 **정산 없음**
- 신청자 0명일 때만 모집 설정 변경 가능
- 초대권 만료 전 **30/7/3/1일** 푸시 알림

### **유저 측**

- 메인 홈 "웰컴 특가" 앵글에서 **구좌 기준**으로 노출 (진입 경로 파라미터로 체험단 여부 분기)
- 같은 모임이라도 일반 목록에서 들어가면 일반 모임, 웰컴 특가에서 들어가면 체험단 모임
- 본인인증 필수 (기존 모달 재사용)
- 체험단 상세 페이지는 **웹뷰로 전환**
- 결제 → 참여 확인 → 리뷰 작성(7일 이내) → **즉시 80% 부분환불** (검수 없음)

### **기술 구조**

- **기존 테이블 확장**: `SocialingMember.grade`에 `TRIAL` 추가, `Order`에 `orderType` 필드 추가
- **정원 합산**: 일반 + 체험단 공유 (별도 정원 아님)
- **승인제 독립**: 소셜링 승인제와 체험단 승인제 각각 별개 운영
- **부분환불**: 기존 얼리버드 `EarlyBirdRefundService` + `BootpayService.refund` 패턴 활용 가능 (차이점: 모임 완료 시 일괄이 아닌 리뷰 작성 즉시 개별 처리)
- **범위**: 백엔드(munto-backend) + 웹 프론트(소셜링 상세 웹뷰) + 모바일(munto-mobile) 전체

### **미정/대기 사항**

- **호스트 참여 확인 방법**: QR POC 진행 중 ([APPF-797](https://munto.atlassian.net/browse/APPF-797))
- **초대권 환불 화면**: 디자인 내일 업데이트
- **체험단 설정 변경 화면**: 기획에서 추가 예정
- **엣지케이스**: 기획에 요청 예정

## 2. 소셜링 상세 페이지

### 개발 결정 사항

- 체험단 대상 유저에게는 상세 페이지에서 할인가, 체험단 안내 등을 분기하여 표시합니다.
- 체험단 전용 안내 섹션이 별도로 추가됩니다.

### 개발 제안 (논의 요청)

현재 앱의 소셜링 상세 페이지는 기술 부채가 심각하여, 체험단 분기를 추가하는 데 2주 정도 소요될 것으로 예상됩니다. 반면 웹 프론트엔드 상세 페이지는 구조가 깔끔하고, 디자인도 최신 버전이 적용되어 있어 1주 이내로 체험단 분기 추가가 가능합니다.

**제안 선택지:**

- **A) 전체 소셜링 상세를 웹뷰로 전환 + 디자인 최신화**
    - 체험단 개발과 함께 상세 페이지 전체를 웹 기반으로 변경
- **B) 체험단 유저에게만 웹뷰 상세 적용 → A/B 비교**
    - 일반 모임(기존 앱) vs 체험단 디자인 적용 모임(웹) 구매율 비교 후 전체 전환 여부 판단
- **C) 기존 소셜링 상세 페이지를 활용 + 체험단 분기**
    - 기존 앱 화면 그대로 활용.

### 확인 요청

**Q2-1. 상세 페이지 웹뷰 전환**
위 제안에 대해 어떻게 생각하시나요?

- A) 전체 웹뷰 전환 + 디자인 최신화
- B) 체험단만 웹뷰 → A/B 테스트 후 판단
- C) 기존 앱 상세 페이지 유지 (개발팀에서 리팩토링)

개발팀에서는 B안을 추천합니다. 디자인 최신화 의향이 있다면 체험단 개발과 함께 진행하는 것이 효율적입니다.

**Q2-2. 디자인 최신화 의향**
현재 웹 디자인 기준으로 앱 상세 페이지 디자인도 최신화할 계획이 있으신가요?

**Q2-3. 가격 표시**
같은 소셜링 상세 페이지에서 유저 유형에 따라 다르게 보이나요?

- 일반 유저: 원가만 표시
- 체험단 유저: 원가 + 할인가 + 환급 안내
- 체험단 유저 + 얼리버드: 미정

**Q2-4. 참여 인원 표시**
일반/체험단 자리 구분이 유저에게 보여야 하나요? (예: "일반 6/6 마감 · 체험단 2/4 남음")

**Q2-5. 얼리버드 + 체험단 중복**
한 소셜링에 얼리버드 할인과 체험단 할인이 동시에 적용될 수 있나요? 있다면 가격 표시 우선순위는?

## 6. 정산

### 개발 결정 사항

- 체험단 정산은 기존 정산과 완전히 분리하여 운영합니다.
    - 기존 정산: 모임 종료 후 실행 (일반 멤버 결제분)
    - 체험단 정산: **모임 종료 + 7일 후** 실행 (리뷰 환급 확정 이후)
- 이렇게 하면 리뷰 기한과 정산 시점이 충돌하지 않습니다.

## 2차 문의사항

호스트 참여 확인 방법: 체험단 멤버의 모임 참여를 호스트가 확인하는 방식이 버튼/QR 중 미정 상태입니다. 리뷰 작성 가능 조건(참여 확인 완료 후 리뷰 작성 → 즉시 환급)에 영향을 주기 때문에, 방식과 화면을 확정해주세요.

- 금일 중에는 화면 작업 시간이 없어서, 간단한 플로우만 전달드립니다…([피그마 링크](https://www.figma.com/design/fCdsDUQ1tzpTD6ZXFx9HRM/%EC%B2%B4%ED%97%98%EB%8B%A8?node-id=168-5776&t=Id9KJ9DWbKsSbIdo-1))

체험단 설정 변경: 체험단 모집 시작 후 인원 포함 설정 변경이 불가한데, 변경이 필요한 경우 처리 방법을 정해주세요. (예: 모집 닫고 다시 열기 가능? 아예 변경 불가?)

호스트 초대권 유효기간 만료 전 푸시알림이 있나요?

- 고려하지 못한 사항인데 30일, 7일, 3일, 1일 발송되어야할것 같습니다!

초대권 환불 플로우 화면 필요합니다.

- 넵 해당화면 추가하겠습니다! 금일 중으로는 어려울거 같아, 내일 중으로 화면은 업데이트 하겠습니다!

백오피스 플로우 필요합니다.

- 해당 플로우는 차주 중에 업데이트 하겠습니다!
