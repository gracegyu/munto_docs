# 기기 UDID 기반 접속 제한 — 정책 케이스 매트릭스

- **기준 문서:** `기기 UDID 기반 접속 제한 정책.md` 본문(`## 1. 배경 및 목적` 이후)
- **목적:** 정책의 모든 판정 규칙을 **축(다계정 / 다기기) × 상태(활성 / 미사용 / 탈퇴)** 로 빠짐없이 정리하여, OnePager·개발·QA의 기준으로 사용
- **작성일:** 2026-03-23

---

## 전제 조건

| 항목 | 내용 |
|------|------|
| 적용 범위 | 네이티브 앱(iOS IDFV / Android ANDROID_ID). 웹 브라우저는 UDID 미수집으로 **정책 대상 외**. |
| 기기 카운트 윈도 | **최근 90일** 이내 로그인 이력이 있는 기기만 슬롯에 포함. 90일 초과 미사용 기기는 집계에서 자동 제외. |
| 계정 카운트 윈도 | **기간 제한 없음.** 해당 기기에서 로그인한 계정 이력 전체가 기준. |
| 탈퇴 유예 | 동일 전화번호 기준 **탈퇴 후 30일** 이내 신규 가입 불가(기존 도메인 규칙). |

## 축 1. 단일 기기 — 다중 계정 (1기기 최대 2계정)

정책 근거: `## 3-1`, `## 2-3(계정 카운트 무기한)`

| # | 시나리오 | 현재 슬롯 | 시도 | 결과 | 비고 |
|---|----------|-----------|------|------|------|
| 1-1 | 기기 D에 계정 A만 연결 | 1/2 | B가 D에서 로그인 | ✅ 허용 | 정상 범위 내 |
| 1-2 | 기기 D에 A, B 연결 (둘 다 활성) | 2/2 | C가 D에서 로그인/가입 | ❌ 차단 | `DEVICE_ACCOUNT_LIMIT` |
| 1-3 | 기기 D에 A, B 연결, **A가 90일 넘게 미사용** | 2/2 | C가 D에서 로그인/가입 | ❌ 차단 | **계정 카운트는 무기한** → 미사용이어도 슬롯 유지 |
| 1-4 | 기기 D에 A, B 연결, **A가 탈퇴** (30일 이내) | 탈퇴 유예 | C가 D에서 가입 (A와 동일 번호) | ❌ 차단 | 전화번호 탈퇴 유예(30일) 우선 적용 |
| 1-5 | 기기 D에 A, B 연결, **A가 탈퇴** (30일 경과, 슬롯 삭제 완료) | 1/2 | C가 D에서 가입 | ✅ 허용 | A의 슬롯이 삭제되어 B만 남음 → 여유 |
| 1-6 | 기기 D에 A, B 연결, **A가 탈퇴** (30일 경과, 슬롯 미삭제) | 2/2 | C가 D에서 가입 | ❌ 차단 | 슬롯 삭제 배치가 안 돌았으면 여전히 2/2 |
| 1-7 | 기기 D에 A, B 연결, **B가 다른 기기 D2로 이동** 후 D에서 90일 넘게 미사용 | 2/2 | C가 D에서 로그인 | ❌ 차단 | 계정 카운트는 무기한이므로 B가 D에서 안 써도 슬롯 유지 |

## 축 2. 단일 계정 — 다중 기기 (1계정 최대 2기기)

정책 근거: `## 3-2`, `## 2-3(기기 카운트 90일)`

| # | 시나리오 | 현재 슬롯 | 시도 | 결과 | 비고 |
|---|----------|-----------|------|------|------|
| 2-1 | 계정 A가 기기 D1에서만 사용 | 1/2 | A가 D2에서 로그인 | ✅ 허용 | 정상 범위 내 |
| 2-2 | 계정 A가 D1, D2에서 사용 (둘 다 90일 내 활성) | 2/2 | A가 D3에서 로그인 | ❌ 차단 | `ACCOUNT_DEVICE_LIMIT` |
| 2-3 | 계정 A가 D1, D2에서 사용, **D1이 90일 넘게 미사용** | 1/2 | A가 D3에서 로그인 | ✅ 허용 | 기기 카운트 90일 → D1이 집계에서 빠져 D2만 남음 |
| 2-4 | 계정 A가 D1, D2에서 사용, **D1이 89일 미사용** | 2/2 | A가 D3에서 로그인 | ❌ 차단 | 아직 90일 미도래 → D1 슬롯 유지 |
| 2-5 | 계정 A가 D1, D2에서 사용, **A가 탈퇴** | — | — | — | 탈퇴 시 모든 기기 바인딩에 소프트 삭제 → 30일 후 하드 삭제 |
| 2-6 | 계정 A가 D1, D2에서 사용, **D1 공장 초기화** (ANDROID_ID 변경) | 1/2 (기존 D1 식별자 기준) | A가 D1'(새 ID)에서 로그인 | 2/2 → 허용 | 새 ID는 별도 기기로 인식. 기존 D1 바인딩은 90일 후 자동 제외. |

## 축 3. 탈퇴 × 기기 슬롯

정책 근거: `📌 1`, 탈퇴 30일 유예

| # | 시나리오 | 결과 | 비고 |
|---|----------|------|------|
| 3-1 | 기기 D에 A, B. A 탈퇴 → **30일 이내**, 동일 번호로 C 가입 시도 | ❌ 차단 | 전화번호 탈퇴 유예 |
| 3-2 | 기기 D에 A, B. A 탈퇴 → **30일 이후**, 동일 번호로 C 가입 시도 | ✅ 허용 (슬롯 삭제 완료 시) | A 슬롯 하드 삭제 → D는 B만 → C 가입 가능 |
| 3-3 | 기기 D에 A, B. A 탈퇴 → **30일 이후**, 다른 번호로 C 가입 시도 | ✅ 허용 (슬롯 삭제 완료 시) | 전화번호 유예와 무관 |
| 3-4 | 기기 D에 A, B. **A, B 둘 다 탈퇴** → 30일 이후 | 0/2 | D의 슬롯 비어 있음 → 신규 가입 가능 |

## 축 4. 교차 케이스 (다계정 + 다기기 동시)

| # | 시나리오 | 결과 | 비고 |
|---|----------|------|------|
| 4-1 | 계정 A가 D1, D2 사용(2기기). 기기 D1에 A, B 연결(2계정). C가 D1에서 로그인 시도. | ❌ 차단 | D1 기준 계정 슬롯 초과 (`DEVICE_ACCOUNT_LIMIT`) |
| 4-2 | 계정 A가 D1, D2 사용(2기기). A가 D3에서 로그인 시도. D1은 90일 넘게 미사용. | ✅ 허용 | 기기 카운트 90일 → D1 제외 → A는 D2만 활성 → D3 가능 |
| 4-3 | 계정 A가 D1, D2 사용(2기기). 기기 D1에 A, B 연결(2계정). A가 D3에서 로그인 시도. D1, D2 모두 활성. | ❌ 차단 | A 기준 기기 슬롯 초과 (`ACCOUNT_DEVICE_LIMIT`) |

## 기존 `📌` 블록 대비 보강된 점

| 기존 `📌`에 있던 것 | 본 매트릭스에서 보강 |
|---------------------|----------------------|
| 탈퇴 30일 유예 (📌 1) | ✅ 포함 + 슬롯 삭제 완료/미완료 분기 추가 (3-2, 1-5, 1-6) |
| 동시 이용 시 추가 차단 (📌 2) | ✅ 축 1의 1-2로 흡수 (별도 규칙이 아닌 2/2 초과의 자연스러운 결과) |
| 90일 미사용 슬롯 유지 (📌 3) | ✅ **축별로 다름**: 계정 카운트 무기한(1-3) vs 기기 카운트 90일(2-3). 📌 3은 축 1 케이스. |
| 다기기 축 | ❌ 📌에 없었음 → ✅ 축 2 전체 |
| 교차 케이스 (다계정+다기기) | ❌ 📌에 없었음 → ✅ 축 4 |
| 기기 90일과 계정 무기한의 차이 | ❌ 📌에 혼재 → ✅ 전제 조건 + 각 축에서 구분 |
