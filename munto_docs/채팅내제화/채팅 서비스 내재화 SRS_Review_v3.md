## 2. 추가 리뷰 의견

### Comment 1 (P1): 존재하지 않는 섹션 참조 (284번 줄)

284번 줄에서 "상세 내용은 6.1.4 채팅 푸시 알림 아키텍처 참조"라고 되어 있지만, **6.1.4 섹션은 문서에 존재하지 않습니다**. 실제 섹션은 6.1.1(데이터 손실 방지), 6.1.2(사용자 보호), 6.1.3(시스템 안정성)까지만 있습니다.

푸시 알림 아키텍처 상세 내용은 1250번 줄 부근(푸시 알림 SLA, 섹션 번호 없음)과 7.5 푸시 알림(2779번 줄)에 분산되어 있습니다. 올바른 섹션 번호로 수정하거나, 참조 대상 섹션에 번호를 부여해 주세요.

### Comment 2 (P2): Redis Failover 시 푸시 이벤트 유실 대응

Redis Streams 기반 푸시 파이프라인이 잘 정의되어 있습니다. 다만 Redis Primary 장애 → Replica Failover 동안 (수 초~수십 초) XADD가 실패하면 Offline 사용자 푸시가 유실될 수 있습니다.

"best-effort"로 명시되어 있어 허용 범위인지, Chat Server에서 XADD 실패 시 in-memory 큐잉 후 재시도가 필요한지 확인 부탁드립니다.

## 3. 결론

SRS v1.0.2는 이전 리뷰의 핵심 의견(Kafka→Redis Streams, 아키텍처 개선)이 **전부 잘 반영**되어 기술적 완성도가 크게 올라갔습니다. 추가 의견 2건 중 Comment 1(섹션 참조 오류)은 수정이 필요하고, Comment 2는 경미한 수준입니다.
