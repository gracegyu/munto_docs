**[데이팅] 매칭 알고리즘 점수 계산 공식 개선**

## **Description**

| 항목 | 내용 |
| --- | --- |
| **배경** | 대표님과 알고리즘 분석 리포트(2026-01-22) 검토 후 점수 계산 공식 개선 필요성 도출 |
| **현재 문제점** | 1. **점수 분포가 정규분포화 되어 있지 않음** - 남성 평균 0.25, 중앙값 0.15로 왜도가 높음 (정확도 30~43%)
2. **좋아요와 점수 간 상관관계 약함** - 남성 r=0.25, 여성 r=0.16으로 상관계수가 낮음 (r < 0.3)
3. **성별 간 좋아요 비율 불균형** - 여성 34% vs 남성 11% (약 3.2배 차이)인데 평균 점수는 동일(0.13) |
| **개선 방향** | 1. 점수 분포의 정규분포화를 위한 계산식 조정
2. 좋아요 비율이 점수에 더 크게 반영되도록 가중치 재설계
3. 실제 매력도(좋아요 비율)와 점수 간 상관관계 강화
4. 신규 유저가 통계적으로 유의미한 점수를 획득하기 위한 최소 평가 횟수(표본 크기)를 신뢰구간 기반으로 산출 |
| **고려요소** | • 기존 유저 점수의 급격한 변동 최소화
• 신규 유저와 기존 유저 간 점수 형평성
• 성별 간 노출 비율 불균형(남:여 = 1:11.5) 고려
• 점수 변경 후 A/B 테스트 필요 |
| **기대효과** | • 점수의 변별력 향상 (정규분포화)
• 좋아요를 많이 받는 유저가 실제로 높은 점수를 받도록 개선
• 매칭 품질 향상 및 사용자 만족도 증가 |

## 현재 점수 계산 공식

```
(이름을 못정함)점수 = W1×매력도 + W2×활동성 + W3×수익유발 + W4×지출 + 신규유저보너스
최종점수 = W_age×나이점수 + W_loc×지역점수 + W_attr×(이름을 못정함)점수
```
