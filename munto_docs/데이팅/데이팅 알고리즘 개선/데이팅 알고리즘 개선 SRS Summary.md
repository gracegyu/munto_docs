# 데이팅 알고리즘 개선 SRS — 요약

> 원본: `데이팅 알고리즘 개선 SRS.md` (v1.0.0, 1,727줄)
> 작성 목적: 리뷰어가 전체 알고리즘 구조를 빠르게 파악하기 위한 요약

---

## 한 줄 요약

기존 단순 가중합 점수(`매력도×W1 + 활동성×W2 + 수익유발×W3 + 지출×W4`)를 **베이지안 추론 기반 단일 점수 체계**로 교체하고, 성별·분류별로 차별화된 노출 전략을 적용하는 프로젝트입니다.

---

## 1. 핵심 변경 — 기존 vs 변경

| 항목 | 기존 (AS-IS) | 변경 (TO-BE) |
|------|-------------|-------------|
| 점수 체계 | 가중합 4요소 | **베이지안 단일 점수** `(α₀+k)/(α₀+β₀+n)` |
| 신규유저 처리 | 7일간 감소 보너스 | **Prior(사전확률)로 대체** |
| 추천/발견 | 별도 사이클 운영 | **동일 사이클 통합** (하루 3회, 각 8명) |
| 성별 로직 | 가중치만 다름 | **완전히 다른 노출 규칙** (뷰어 7유형) |
| 품질 관리 | 없음 | **노출 제한 정책** (하위 15% + 부정 시그널) |

---

## 2. 유저 분류 (3단계)

모든 유저는 **받은 인터랙션 횟수** 기준으로 분류됩니다.

| 분류 | 남성 기준 | 여성 기준 | 베이지안 특성 |
|------|----------|----------|-------------|
| NEW | 0~4회 | 15회 미만 | Prior가 점수 지배 |
| SETTLING | 5~14회 | 15회+ AND 가입 D+7 이내 | Prior ↔ 관측 균형 |
| ESTABLISHED | 15회+ | 가입 D+7 이후 | 관측 데이터 지배 |

---

## 3. 베이지안 점수 계산

```
score = (α₀ + k) / (α₀ + β₀ + n)

α₀, β₀ = 사전확률 파라미터 (성별별 다름)
k = 긍정 인터랙션 가중치 합
n = 전체 인터랙션 가중치 합
```

| 성별 | Prior | 초기 점수 | 의미 |
|------|-------|----------|------|
| 남성 | Beta(3.0, 7.0) | 0.30 | 보수적 시작 (풀이 크므로) |
| 여성 | Beta(3.5, 6.5) | 0.35 | 약간 높게 시작 (풀 희소성) |

- 긍정 인터랙션(좋아요 0.5, 슈퍼좋아요 1.0 등) → k와 n 모두 증가
- 부정 인터랙션(패스 0.1) → **n만 증가** → 긍정 비율 희석
- 관측이 쌓일수록 Prior는 자연스럽게 희석되어 실제 비율에 수렴

---

## 4. 버킷 시스템 — percentile 정규화

버킷은 노출 경계가 아니라, **점수 비교를 공정하게 하기 위한 Reference Group**입니다.

- **구성**: 연령대(4) x 지역(2) x 성별(2) = 최대 16개 버킷
- **역할**: 동일 버킷 내에서 percentile(0~100)을 계산하여 교차 버킷 비교를 공정하게 함
- **갱신 시점**: raw_score는 실시간, percentile은 사이클 직전 계산

```
예: 20대 남성 raw 0.45 (버킷 내 85%)  vs  30대 남성 raw 0.40 (버킷 내 78%)
    → percentile 기준으로 20대 남성이 상위
```

## 5. 뷰어 7유형 — 누가 누구를 보는가

사이클 실행 시 각 유저는 아래 7개 유형 중 하나로 분류되며, **후보 풀과 슬롯 규칙이 다릅니다**.

| # | 뷰어 | 보는 후보 | 핵심 |
|---|------|----------|------|
| 1 | 기존 남성 (정상) | 기존 + 신규/정착중 여성 | 신규여성 최대 75% |
| 2 | 신규/정착중 남성 | **기존 여성만** | 검증된 여성만 |
| 3 | 기존 여성 (정상) | 기존 + 신규/정착중 남성 | 신규남성 최대 25% |
| 4 | 신규 여성 | 기존 남성 **상위 30%만** | 좋은 첫인상 보장 |
| 5 | 정착중 여성 | 기존 남성 (30%→자기구간 점진) | D+7까지 점진 이동 |
| 6 | 노출 제한 남성 | **제한 여성만** | 하드 격리 |
| 7 | 노출 제한 여성 | 유형 3과 동일 (정상) | 피드는 정상 유지 |

---

## 6. 노출 제한 정책

| 트리거 | 조건 | 적용 |
|--------|------|------|
| 하위 점수 | percentile 하위 15% 이하 | 사이클 시 자동 판정 |
| 신고/차단 | 1건 | 즉시 |
| 빠른 패스 | impression 후 3초 이내 DISLIKE 3회 누적 | 3회 도달 시 |
| 첫 노출 전원 미조회/패스 | 첫 노출 3명 전원 | 즉시 |

**비대칭 설계**: 제한 남성은 하드 격리(제한 여성만 봄), 제한 여성은 정상 피드 유지(여성 풀 희소성 보호)

**복귀 경로**: 프로필 변경이 유일한 능동적 복귀 수단 (7일 내 1회 제한)

---

## 7. 사이클 운영 — 후보 선택 파이프라인

```
하루 3회 (00:00, 08:00, 16:00), 매 사이클 = 추천 4명 + 발견 4명

[Phase 1] 버킷별 워커 16개 병렬
  → DB에서 유저 로드 → percentile 계산 → 서브 풀 3분할
    (established / new_settling / restricted)

[Phase 2] 유저별 단일 잡 (서버 메모리, DB 접근 없음)
  Step 2. 뷰어 유형 판별 (7유형)
  Step 3. 기본 제외 (이미 본 유저, 차단, 비활성 등)
  Step 4. 교차 노출 규칙 (신규끼리 못봄)
  Step 5. 유저 설정 필터 (나이, 학력, 흡연 등)
  Step 6. 가중치 산출 + 확률 샘플링 → 8명 추출
```

**가중치 산출 수식**:
```
weight = percentile x 나이차점수 x (1 + MBTI가점) x (1 + 키워드가점) x ...
```

**샘플링**: 가중 랜덤(Weighted Random without Replacement) — top-N이 아닌 확률적 추출

## 8. 유저 상태 변경 시 처리

| 상태 변경 | 분류 | 점수 | 인터랙션 | 노출 제한 |
|-----------|------|------|----------|----------|
| 비활성화 후 해제 | 유지 | 유지 | 유지 | 유지 |
| 프로필 변경 | SETTLING 전환 | Prior 초기화 | 전체 삭제 | 무조건 해제 |
| 탈퇴 후 재가입 | NEW 초기화 | Prior 초기화 | 전체 삭제 | 해제 |

## 9. 운영 파라미터 (system-config)

모든 주요 파라미터는 코드 배포 없이 동적 조정 가능합니다.

- Prior 파라미터 (α₀, β₀)
- 인터랙션 가중치
- 노출 제한 임계값 (하위 %, 빠른 패스 횟수/시간)
- 사이클 실행 시각, 카드 수
- 버킷 연령대 경계, 지역 분류
- 미접속 유저 제외 기준일
- 나이차 점수표, 가점 수치 등
