# [취향인연] 데이팅 앱 독립 분리 준비

## 배경

데이팅 기능을 문토 앱에서 독립된 별도 앱으로 분리할 예정입니다. 기획 확정 전에 인프라 및 앱 등록 등 분리를 위한 사전 준비 작업을 진행합니다.

## 현재 문제점

- 데이팅 앱이 문토 앱 내 패키지 형태로 동작하고 있어 Firebase 프로젝트, 스토어 앱 등이 분리되어 있지 않습니다.
- Analytics, FCM, Remote Config 데이터가 문토와 혼재될 수 있습니다.

## 개선 방향

1. Firebase 프로젝트 분리 (FCM, Analytics, Remote Config) 및 google-services.json / GoogleService-Info.plist 교체
2. App Store Connect 데이팅 앱 신규 등록 (번들 ID: kr.munto.dating)
3. Google Play Console 데이팅 앱 신규 등록
4. dating-backend Firebase 프로젝트 분리 (FCM 푸시 독립)
5. Sendbird / Bootpay 웹훅 URL 데이팅 전용으로 변경 (munto-backend 경유 제거)

## 고려요소

- 인증(JWT) 구조는 기존 문토 토큰 호환 방향으로 별도 이슈(WEBB-1196)에서 결정 예정이므로 본 이슈에서는 다루지 않습니다.
- 데이팅 DB User.id = 문토 userId FK 통합 구조는 그대로 유지합니다.
- 디자인 시스템(muntorial) 교체 및 신규 기능은 기획 확정 후 별도 이슈로 진행합니다.
- 딥링크의 경우 문토 앱에 구현된 딥링크 구조를 참고하여 데이팅 앱에도 동일하게 적용 가능한지 검토가 필요합니다. (도메인, App Links / Universal Links, 푸시 알림 딥링크 등)

## 기대효과

- 기획 확정 즉시 데이팅 앱 독립 출시가 가능한 상태로 준비됩니다.
- Firebase 데이터가 앱별로 분리되어 분석 정확도가 향상됩니다.
- 스토어 앱 슬롯 선점으로 출시 일정 리스크가 감소합니다.

- Firebase 프로젝트 분리 (Analytics, FCM, Remote Config)

- 백엔드 웹훅 URL 데이팅 전용으로 전환

**보류 작업 (기획 확정 후 진행)**

- 앱 이름, 로고, 브랜드 컬러 등 디자인 관련

인프라 분리는 기획 내용과 무관하게 진행 가능한 작업들이라 먼저 착수하면 기획 확정 후 빠르게 출시 준비가 가능할 것으로 판단됩니다.
