# 문토-데이팅 본인인증 스냅샷 필드

## **1. Bootpay 검증 직후 각 도메인 상태 분석**

Bootpay `certificate` 응답을 기준으로 보면, **동일한 원천 데이터**를 두 제품이 **서로 다른 테이블과 타입**으로 나누어 저장합니다.

| 항목 (Bootpay) | 문토 (`munto`) | 데이팅 (`munto_dating`) |
| --- | --- | --- |
| `authenticate_data.unique` → 해시 | `PersonalAuthentication.hashedCI`(SHA-256→base64, `verify`와 동일한 식) | `IdentityVerification.hashedCI`(동일한 식) |
| 이름 | `PersonalAuthentication.name`에 저장합니다. 동일 트랜잭션에서 `User.name`은 **갱신하지 않습니다**(`verify` 코드 기준). | `User.name`에 저장합니다. `IdentityVerification`에는 이름 컬럼이 없습니다. |
| 생년 문자열 | `PersonalAuthentication.birth`(문자열 그대로)에 저장합니다. 동시에 `User.birthDay`를 `dayjs(birth).toDate()`로 맞춥니다. | `User.birthDate`만 저장합니다(문자열을 파싱한 뒤 `Date`로 변환). `IdentityVerification`에는 생일 컬럼이 없습니다. |
| 성별 | `PersonalAuthentication.gender`(숫자)에 저장합니다. 동시에 `User.sex`를 `gender === 1`이면 MALE, 아니면 FEMALE로 설정합니다. | `User.gender`만 저장합니다(`Gender` enum, Bootpay 값을 boolean 식으로 매핑). `IdentityVerification`에는 없습니다. |
| 전화 | `PersonalAuthentication.phone`에 Bootpay 전화번호(문자열)를 저장합니다. `User.phoneNumber`는 **가입 시 등록 번호와의 일치 여부만 검증**하며, 본 트랜잭션에서는 **갱신하지 않습니다**. | `User.phoneNumber`에만 저장하며 **`CryptoUtil.encryptPhoneNumber`**를 적용합니다. `IdentityVerification`에는 없습니다. |

**정리:** 문토는 본인인증 전용 행인 **`PersonalAuthentication`에 이름, 생년월일 문자열, 성별 숫자, 평문 전화번호, 해시를 함께 둡니다.** 데이팅은 **해시만 `IdentityVerification`에 두고**, 나머지 개인정보는 **`User`에만** 저장하며, **전화번호는 암호화하여 저장합니다.** 이에 따라 스냅샷으로 **공통 DTO**를 만든 뒤, 후행 데이터베이스에는 **데이팅 스키마에 맞게 분배하거나 문토처럼 `PersonalAuthentication` 한 테이블에 합쳐** 넣어야 합니다.

`hashedCI` 알고리즘은 양쪽 코드 모두 `sha256(unique).digest('base64')`로 구현되어 있으며, 동치 검증의 전제가 됩니다.

## **2. ① 문토 본인인증 → 데이팅 본인인증**

스냅샷은 `PersonalAuthentication`과 JWT의 `userId` 맥락에서 생성합니다. 데이팅 후행 저장 시에는 아래 표와 같이 **데이팅 스키마**에 맞춥니다.

| 문토 (읽기) | 데이팅 (쓰기) |
| --- | --- |
| `PersonalAuthentication.name` | `User.name` |
| `PersonalAuthentication.birth` | `User.birthDate`(문자열 → `DateTime` 규칙은 합의에 따름) |
| `PersonalAuthentication.gender` | `User.gender`(정수 → `Gender` 매핑은 합의에 따름) |
| `PersonalAuthentication.hashedCI` | `IdentityVerification.hashedCI` |
| `User.id`(JWT 주체와 동일) | `IdentityVerification.userId` |

---

## **3. ② 데이팅 본인인증 → 문토 본인인증**

스냅샷은 `User`와 `IdentityVerification`으로 구성합니다. 문토 후행 반영 시에는 **`PersonalAuthentication`에 Bootpay 응답과 동일한 성격의 필드를 채우고**, 문토 `User`의 외래키 및 정합을 맞춥니다.

| 데이팅 (읽기) | 문토 (쓰기) |
| --- | --- |
| `User.name` | `PersonalAuthentication.name`(필요 시 문토 `User.name` 정책은 별도로 둡니다). |
| `User.birthDate` | `PersonalAuthentication.birth`(문자열 포맷은 합의에 따름), `User.birthDay` |
| `User.gender` | `PersonalAuthentication.gender`, `User.sex`(매핑은 합의에 따름) |
| `IdentityVerification.hashedCI` | `PersonalAuthentication.hashedCI` |
| `User.id`(M2M 경로의 `datingUserId`) | OAuth·연계 저장소로 문토 `User`를 특정한 뒤 `personalAuthenticationId`를 해당 `PersonalAuthentication`에 연결합니다. |
