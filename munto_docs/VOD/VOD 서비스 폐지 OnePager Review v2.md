## 2. v2 변경 사항에 대한 리뷰

### Comment 1 (P0): 환불 처리 시나리오 여전히 누락

VOD 폐지 시나리오 문서에서 가장 큰 비중을 차지하는 것이 환불 처리이고, 폐지 프로세스 구축 문서에서도 수강 중 사용자 구제 방안(환불, 크레딧 보상, 유예 기간)이 상세히 정의되어 있습니다.

OnePager에는 다음이 여전히 빠져 있습니다:
- Risk Assessment에 환불 처리 지연 리스크
- 마일스톤에 환불 처리 운영 체계 구축 일정
- 취소/환불 API 유지 기간 (현재 "5단계 이후까지 유지"로만 기재)
- 환불 유형별 처리 방식 (전액 환불 vs 비례 환불)

환불 처리는 법적 의무(전자상거래법 제17조·제18조)이므로, 최소한 참조 링크나 요약이라도 추가해야 합니다.

### Comment 2 (P0): 약관 개정 일정 마일스톤 누락

폐지 프로세스 구축 문서에서 약관 처리를 **전체 일정의 크리티컬 패스**(Phase 1, 약 2~3주)로 지정하고 있습니다. 현재 마일스톤에서 2단계(4/16) 공지 전에 약관 개정 고지 및 이의 제기 기간 확보가 필요한데, 이 일정이 빠져 있습니다.

약관 변경 시 불리한 변경인 경우 최소 30일 전 고지가 필요합니다. 현재 일정(4/16 공지)으로는 이미 시간이 부족할 수 있으므로, 약관 개정 여부와 일정을 확인해 주세요.

### Comment 3 (P1): 1단계 화면 제거 범위의 삭제 범위 확인

섹션 4의 1단계 화면 제거 범위가 매우 상세하게 작성되어 좋습니다. 다만 몇 가지 확인이 필요합니다:

- `lib/viewmodel/mypage/order/VODApplyViewModel.ts` — **파일 삭제**로 되어 있는데, 이 파일을 참조하는 다른 코드가 있으면 빌드 에러가 발생합니다. import 참조 확인 필요
- `lib/repository/OrderRepository.ts` — 메서드 제거 시, 해당 메서드를 사용하는 다른 ViewModel이나 페이지가 없는지 확인 필요
- 앱(Flutter) `lib/screens/vod_apply/` 폴더 전체 삭제 — 라우트 상수 제거와 함께 `GoRouter` 설정에서도 제거되어야 하는데, `vod_routes.dart`에서만 처리하면 되는지, `app_router.dart` 등 상위 라우터 설정도 확인 필요

### Comment 4 (P1): 제휴 호스트 계약 종료 절차 미기재

2단계에 "호스트 선공지"가 포함되었으나, 폐지 프로세스 구축 문서 Phase 3의 **제휴 호스트 계약 종료 절차**(계약서 전수 확인, 법무 검토, 개별 합의, 위약금 확인)에 대한 언급이 없습니다. 이 부분은 OnePager의 범위 밖이라면 명시적으로 "제휴 호스트 계약은 별도 프로세스로 진행"이라고 기재해 주세요.

### Comment 5 (P1): 폐지 후 인프라 유지/해제 구분

섹션 6 인프라 변경 계획이 DB/마이그레이션 중심으로 작성되어 있어, AWS 인프라 처리가 빠져 있습니다:

- **5단계 이후 해제 가능**: MediaConvert (인코딩), Lambda (인코딩 트리거)
- **5단계까지 유지 필요**: S3 (영상 원본), CloudFront (스트리밍 CDN) — 마이페이지 시청 기능 유지 기간 동안 비용 계속 발생
- **즉시 해제 가능**: 없음 (1~4단계에서는 모든 인프라 유지)

이 구분을 인프라 변경 계획에 추가하면 비용 관리에 도움이 됩니다.

### Comment 6 (P2): 5단계(마이페이지 시청 중단) TBD 구체화

5단계가 TBD로 되어 있는데, Business and Marketing Justification에서 "최초 공지 시 마이페이지 유지 기간을 명확히 기재"라고 Risk Assessment에 적시하고 있습니다. 2단계(4/16) 공지 시점에 종료 예정일을 안내해야 하므로, 5단계 시점을 최소한 "2026년 X월" 수준으로라도 확정해야 합니다.

### Comment 7 (P2): 변경 이력 v1.0.1 변경 내용이 "Technical Description 수정"으로만 기재

v1.0.1에서 Technical Description이 대폭 추가되었는데(API 목록, Course 상태 전이, 마이페이지 처리, 화면 제거 범위, 캐시, 인프라, ERD 7개 섹션), 변경 이력에 "Technical Description 수정"이라고만 되어 있어 변경 범위를 알기 어렵습니다. 주요 추가 항목을 나열해 주세요.

## 4. 종합 평가

v2에서 Technical Description이 대폭 강화되어, **개발 실행에 필요한 기술적 상세**(API 목록, 상태 전이, 화면 제거 범위, 캐시, ERD)가 잘 갖춰졌습니다. 특히 섹션 2(Course 상태 전이)와 섹션 3(마이페이지 시청 처리)은 `CourseEnrollment` 기반 로직까지 코드 수준으로 명시되어 있어, 이전 리뷰에서 요청한 핵심 기술 의견이 충실히 반영되었습니다.

다만 **비즈니스/법적 영역**(환불, 약관 개정, 제휴 호스트 계약)은 v1에서 지적한 사항이 여전히 미반영 상태입니다. 이 부분은 OnePager의 Technical Description 범위 밖이라 의도적으로 제외한 것일 수 있으나, 그렇다면 "해당 내용은 VOD 폐지 시나리오/프로세스 문서 참조"라고 명시적으로 기재하여 문서 간 역할을 명확히 해 주세요.
