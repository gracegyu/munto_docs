# VOD 서비스 폐지 프로세스


| 항목 | 내용 |
| --- | --- |
| 배경 | VOD 폐지 시나리오([Notion—The AI workspace that works for you.](https://www.notion.so/32fe2bc7639d8031abe2e29b2186f9ab?pvs=21) )에 따라, 경영상의 이유로 VOD 서비스를 폐지합니다. |
| 현재 문제점 | VOD 신규 결제가 계속 허용되고 있으며, 앱/웹 전반에 VOD 진입 경로(탐색 페이지, 메뉴, 추천 피드 등)가 다수 존재합니다. 기존 구매자 시청 권한은 유지해야 하므로 단순 삭제가 아닌 세분화된 처리가 필요합니다. |
| 개선 방향 | **1단계 (4/9~4/15): 신규 결제 차단** 웹 구매하기 버튼 제거, VOD 구매 페이지 404 처리, 구매 관련 API 410 Gone 또는 404 응답 처리 (VOD가 웹뷰 기반이므로 웹 수정만으로 앱에도 반영됨)<br><br>**2단계 (4/16): 서비스 종료 공지** 앱/웹 공지 배너 게시, 유료 구매자 이메일/PUSH 개별 통지<br><br>**3단계 (5/18~): VOD 관련 페이지·기능 전면 제거** 앱/웹 VOD 탐색 페이지·메뉴·추천 피드 제거, 호스트 어드민 VOD 관리 기능 비활성화, 마이페이지 '구매한 VOD' 시청 기능은 기존 구매자 한정 유지<br><br>**4단계 (상시): 거래 기록 보존** VOD 구매·결제 기록은 DB에서 물리 삭제 없이 5년간 보존 (전자상거래법 제6조) |
| 고려요소 | 마이페이지 시청 기능은 구매 이력 존재 여부 기준으로 권한을 분리 관리해야 함.<br>구매 API 차단 시 마이페이지 조회 API는 영향 없도록 분리 처리 필요.<br>결제 차단 이후에도 환불 처리를 위한 CS 운영 체계 유지 필요. |
| 기대효과 | 서비스 종료 이후 신규 결제 발생 차단으로 CS 리스크 제거. 법적 의무(약관 제10조 제4항, 전자상거래법 제6조) 준수. 단계적 진행으로 기존 구매자 불편 최소화. |
