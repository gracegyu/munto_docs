# [VOD] 폐지 프로세스 구축

담당부서: 프로덕트
진행상황: 진행중
작업상황: 준비 중
우선순위: 높음

## 배경

문토 플랫폼에서 운영 중인 VOD(영상 콘텐츠) 서비스를 폐지한다. 핵심 서비스에 집중하기 위해 VOD 관련 기능을 전면 제거하며, 호스트 고지부터 기술적 제거, 데이터 정리까지 전체 프로세스를 아래에 정리한다.

---

## 1. VOD 관련 약관 처리

VOD 서비스 폐지에 앞서, 이용약관 내 VOD 관련 조항을 정리하고 약관 개정을 선행

### 1-1. 현행 약관 내 VOD 관련 조항 확인

- [ ]  이용약관 전문에서 VOD 관련 조항 전수 조사
- [ ]  환불·취소 정책 내 VOD 관련 규정 확인
- [ ]  개인정보처리방침 내 VOD 관련 수집·이용 항목 확인
- [ ]  호스트 이용약관 내 VOD 관련 조항 확인

### 1-2. 약관 개정안 작성

- [ ]  VOD 관련 조항 삭제 또는 "서비스 종료" 부칙 추가
- [ ]  서비스 종료에 따른 사용자 권리 보호 조항 반영 (환불, 유예 기간 등)

### 1-3. 약관 개정 고지

전자상거래법·약관규제법에 따라 약관 변경 시 사전 고지가 필요하다.

| 항목 | 내용 |
| --- | --- |
| 고지 시점 | 시행일 최소 7일 전 |
| 고지 채널 | 앱 내 공지사항, 이메일, 앱 내 팝업 |
| 고지 내용 | 변경 전 약관 / 변경 후 약관 비교, 시행일, 이의 제기 방법 |
| 별도 페이지 | 현행 약관 링크 + 개정 약관 별도 페이지 링크 (홈페이지 게시 링크가 아닌 별도 URL) |

### 1-4. 예상 소요 기간

| 작업 | 예상 소요 |
| --- | --- |
| 약관 내 VOD 조항 전수 조사 | 1~2일 |
| 약관 개정안 작성 | 2~3일 |
| 법무팀 최종 검토 | 2~3일 |
| 현행/개정 약관 별도 페이지 생성 (WEBF) | 1~2일 |
| 약관 개정 고지 발송 (이메일 + 앱 공지) | 1일 |
| 이의 제기 기간 확보 | 최소 7일 (불리한 변경 시 30일) |

> ⚠️ **핵심**: 약관 개정 고지 → 이의 제기 기간 확보 → 시행의 순서를 반드시 준수해야 법적 리스크를 줄일 수 있음.
>

## 2. VOD 호스트 고지

VOD 서비스에 등록된 호스트를 파악하고 폐지 사실을 사전 고지한다.

### 2-1. 호스트 목록 추출 및 분류

- [ ]  VOD 콘텐츠를 등록한 전체 호스트 목록 추출
- [ ]  활동 상태 분류: 활성 / 비활성 / 제휴
- [ ]  제휴 호스트 별도 플래그 (Phase 3에서 별도 처리)

### 2-2. 고지 내용

- [ ]  폐지 일정 및 사유
- [ ]  콘텐츠 처리 방안 (삭제 일정, 백업 방법)
- [ ]  정산 관련 안내 (미정산 수익 처리)
- [ ]  Q&A 채널 안내

### 2-3. 고지 채널 및 예상 소요 기간

| 작업 | 예상 소요 |
| --- | --- |
| 호스트 목록 추출  | 반나절 |
| 고지 내용 작성 | 1일 |
| 1차 고지 발송 (이메일 + 앱 알림) | 1일 |
| 리마인더 발송 | 1차 고지 후 약 5일 뒤 |
| 개별 문의 대응 | 고지 발송 후 지속 (약 1~2주) |

## 3. 제휴 호스트 계약 확인 및 마무리

별도 계약이 존재하는 제휴 호스트에 대해 법무 검토와 합의 절차를 진행한다.

- [ ]  제휴 계약서 전수 확인 (계약 기간, 해지 조건, 위약금 등)
- [ ]  법무팀과 계약 종료 절차 협의
- [ ]  제휴 호스트별 개별 연락 및 합의
- [ ]  미정산 수익 확정 및 최종 정산
- [ ]  계약 종료 합의서 작성 및 서명

### 예상 소요 기간

| 작업 | 예상 소요 |
| --- | --- |
| 제휴 계약서 전수 확인 | 1~2일 |
| 법무 검토 및 해지 방안 확정 | 2~3일 |
| 제휴 호스트 개별 연락 | 1~2일 |
| 합의 완료 및 서명 | 3~5일 (호스트 반응에 따라 변동) |

## 4. 수강 중인 사용자 파악 및 구제 방안

VOD를 구매·수강 중인 사용자를 파악하고 적절한 구제 방안을 마련한다.

### 4-1. 사용자 파악

- [ ]  VOD 구매 이력 전체 추출
- [ ]  최근 수강 활동 사용자 분류 (최근 30일/90일 내 시청 기록)
- [ ]  수강 완료 / 미완료 구분
- [ ]  구매 금액·진행률 데이터 추출

### 4-2. 구제 방안

| 방안 | 설명 | 적용 대상 |
| --- | --- | --- |
| 콘텐츠 다운로드 기간 제공 | 폐지 전 일정 기간 동안 구매 콘텐츠 다운로드 허용 | 구매 사용자 전체 |
| 환불 처리 | 미수강 또는 수강 미완료 건에 대한 환불 | 수강 미완료 사용자 |
| 대체 보상 (크레딧·이용권) | 소셜링 할인 쿠폰, 문토 크레딧 등으로 보상 | 구매 사용자 중 환불 불가 건 |
| 유예 기간 운영 | 폐지 후에도 일정 기간 기존 콘텐츠 접근 허용 | 수강 중 사용자 |

### 4-3. 예상 소요 기간

| 작업 | 예상 소요 |
| --- | --- |
| 구매/수강 데이터 추출 | 반나절~1일 |
| 구제 방안 확정 (법무·재무 협의) | 2~3일 |
| 사용자 고지 발송 | 1일 |

## 6. VOD 카테고리 없애기

앱, 웹, 백오피스, 백엔드에서 VOD 관련 기능을 전면 제거한다.

### 프로젝트별 작업

| 프로젝트 | 주요 제거 항목 |
| --- | --- |
| **APPF** (Flutter) | VOD 탭, VOD 상세 화면, VOD 관련 네비게이션, 딥링크 |
| **WEBF** (웹 프론트엔드) | VOD 페이지, VOD 카테고리 필터, VOD 검색 |
| **BACK** (백오피스) | VOD 관리 메뉴, 콘텐츠 등록/수정 화면, VOD 통계 |
| **WEBB** (백엔드) | VOD API 엔드포인트, DB 테이블 (soft delete), 스트리밍 관련 인프라 |

### 예상 소요 기간

| 작업 | 예상 소요 |
| --- | --- |
| 앱(APPF) VOD 기능 제거 | 2~3일 |
| 웹(WEBF) VOD 페이지 제거 | 1~2일 |
| 백오피스(BACK) VOD 관리 제거 | 1~2일 |
| 백엔드(WEBB) API 정리 | 1~2일 |
| 통합 QA | 1~2일 |

> 💡 **전체 소요**: 약 3~5일. 프로젝트별 병렬 작업 가능하며, QA는 개발 완료 후 진행.
>

## 호스트 고지 및 사용자 대응 상세

### 호스트 고지 시나리오

호스트 유형에 따라 고지 방식과 내용을 차등 적용한다.

| 호스트 유형 | 고지 방식 | 고지 내용 | 대응 수위 |
| --- | --- | --- | --- |
| **제휴 호스트** (별도 계약) | 개별 전화 + 이메일 + 대면 미팅 (필요 시) | 계약 종료 절차, 최종 정산, 위약금 여부, 대안 제시 | 🔴 최우선 |
| **활성 호스트** (최근 3개월 내 콘텐츠 업로드/수익 발생) | 개별 이메일 + 앱 알림 | 폐지 일정, 콘텐츠 백업 안내, 정산 안내, 소셜링 전환 유도 | 🟡 높음 |
| **비활성 호스트** (3개월 이상 미활동) | 이메일 일괄 발송 | 폐지 일정, 콘텐츠 처리 안내 | 🟢 일반 |

### 호스트 예상 문의 및 대응 스크립트

| 예상 문의 | 대응 방침 |
| --- | --- |
| "왜 갑자기 폐지하나요?" | 핵심 서비스 집중을 위한 전략적 판단임을 안내. 사전 히스토리 문서 참조. |
| "내 콘텐츠는 어떻게 되나요?" | 폐지 전까지 콘텐츠 다운로드 가능 기간을 안내. 백업 방법 가이드 제공. |
| "미정산 수익은 언제 받나요?" | 최종 정산 일정을 구체적으로 안내 (D+○ 이내). |
| "다른 플랫폼으로 이전할 수 있나요?" | 콘텐츠 원본 파일 다운로드 안내. 플랫폼 간 이전은 지원 범위 외. |
| "보상은 없나요?" | 대응 대책 수립(아래 섹션) 기준에 따라 유형별 차등 안내. |

### 사용자(수강생) 대응 시나리오

| 사용자 유형 | 대응 방식 | 대응 내용 |
| --- | --- | --- |
| 수강 완료 사용자 | 이메일 고지 | 서비스 종료 안내, 수강 이력 보존 여부 안내 |
| 수강 중 사용자 (진행률 50% 이상) | 이메일 + 앱 알림 | 유예 기간 안내, 콘텐츠 다운로드 안내 |
| 수강 중 사용자 (진행률 50% 미만) | 이메일 + 앱 알림 | 환불 또는 크레딧 보상 안내 |
| 최근 구매 후 미수강 사용자 | 이메일 + 앱 알림 + 개별 CS 대응 | 전액 환불 안내 |

## 대응 대책 수립 (중요도 기반 차등 대응)

VOD 폐지로 영향 받는 이해관계자를 중요도별로 분류하고, 각 그룹에 맞는 대응 대책을 수립한다. 보상은 금전적 보상에 한정하지 않고, 플랫폼 내 대체 가치를 제공하는 방향으로 설계한다.

### 중요도 판단 기준

| 기준 | 설명 |
| --- | --- |
| 계약적 의무 | 별도 계약이 있는 경우 법적 의무가 우선 |
| 금전적 피해 규모 | 구매 금액, 미정산 수익, 진행률 등 |
| 플랫폼 이탈 리스크 | 해당 이해관계자가 문토를 떠날 가능성 |
| 커뮤니티 영향력 | 다른 유저에게 미치는 영향 (리뷰, 입소문 등) |
| 재전환 가능성 | 소셜링 등 핵심 서비스 이용자로 전환 가능성 |

### 이해관계자별 대응

| 대상 | 중요도 | 대응 대책 | 보상 방안 (비금전 중심) |
| --- | --- | --- | --- |
| **제휴 호스트** | 🔴 최상 | 개별 미팅, 계약 종료 합의, 최종 정산 | 소셜링 호스트 전환 시 수수료 우대 (일정 기간), 호스트 티어 우대, 신규 서비스 우선 참여권 |
| **활성 호스트 (고수익)** | 🔴 높음 | 개별 연락, 전환 지원 | 소셜링 호스트 전환 가이드 + 노출 부스트, 호스트 커뮤니티 초대 |
| **활성 호스트 (저수익)** | 🟡 중간 | 이메일 고지 + 전환 유도 | 소셜링 첫 개설 시 프로모션 지원 (배너 노출 등) |
| **비활성 호스트** | 🟢 낮음 | 이메일 일괄 고지 | 별도 보상 불필요 |
| **수강 중 사용자 (고금액/미완료)** | 🔴 높음 | 개별 CS 대응, 선택지 제공 | 환불 + 소셜링 체험 쿠폰, 콘텐츠 다운로드 기간 충분 확보 |
| **수강 중 사용자 (저금액/거의 완료)** | 🟡 중간 | 이메일 고지 | 유예 기간 제공 + 소셜링 할인 쿠폰 |
| **구매 후 미수강 사용자** | 🔴 높음 | 자동 환불 처리 | 전액 환불 + 소셜링 체험 쿠폰 |
| **수강 완료 사용자** | 🟢 낮음 | 이메일 고지 | 감사 메시지 + 소셜링 할인 쿠폰 (선택) |

### 대책 강구 — 핵심 원칙

1. **법적 의무 최우선**: 제휴 계약, 환불 의무 등 법적으로 반드시 이행해야 할 사항을 먼저 처리한다.
2. **핵심 서비스 전환 유도**: 보상의 방향은 "문토를 떠나지 않도록" 핵심 서비스(소셜링, 취향인연)로의 자연스러운 전환을 유도하는 데 초점을 맞춘다.
3. **비금전적 가치 우선**: 금전적 보상보다 플랫폼 내 특전(노출 부스트, 수수료 우대, 조기 접근권 등)을 활용하여 이탈을 방지한다.
4. **투명한 소통**: 폐지 사유와 대안을 명확하게 전달하여 신뢰를 유지한다.
