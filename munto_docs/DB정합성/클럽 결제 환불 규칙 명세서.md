## 1. 클럽 결제 구조

```
ClubMember (1:1) ClubMembership (1:1) Order (1:1) Payment
```

- 유료 클럽만 ClubMembership이 생성된다
- 무료 클럽은 ClubMembership 없이 Order만 생성된다
- `ClubMembership.billingKey`에 빌링키 저장 (SUBSCRIPTION 타입)

### 결제 방식 분기

```
Club.price > 0  →  빌링키 방식 (Bootpay 빌링키 등록 후 즉시 청구)
Club.price = 0  →  무료 (PG 호출 없음, Order만 생성)
```

### 멤버십 타입별 결제

```
ONETIME       단건 결제 후 영구 멤버십 (expireAt = 2099-12-31)
SUBSCRIPTION  가입 시 1개월치 선결제, 이후 매월 자동 갱신
```

## 2. 결제 확정 플로우

### 2-1. 빌링키 등록 및 즉시 청구 (유료 클럽)

```
[단계 A] 클럽 가용 여부 재검증 (클럽 도메인)
  - availCount > 0
  - Club.deletedAt IS NULL

[단계 B] Bootpay 빌링키 처리
  Bootpay.confirmReceipt(receiptId)로 빌링키 취득
  UserBillingKey 테이블 UPSERT (유저당 1개 활성 키 유지)
  Bootpay.requestSubscribeCardPayment() 즉시 청구

[단계 C] DB 트랜잭션
  Order.orderStatus = WAITING_PAYMENT → COMPLETED_PAYMENT
  OrderHistory 생성
  Payment 생성
  ClubMember 생성/UPSERT
  ClubMembership 생성
    - ONETIME: expireAt = 2099-12-31
    - SUBSCRIPTION: expireAt = now() + 1개월
  Club.availCount 차감 (APPROVE인 경우)

[단계 D] 후처리 (트랜잭션 외부)
  Sendbird 채널 참여
  Push 발송
```

### 2-2. 무료 클럽

```
[단계 A] 클럽 가용 여부 재검증

[단계 B] 이 단계 건너뜀 (PG 호출 없음)

[단계 C] DB 트랜잭션
  Order.orderStatus = WAITING_PAYMENT → COMPLETED_PAYMENT
  OrderHistory 생성
  ClubMember 생성/UPSERT
  Club.availCount 차감 (APPROVE인 경우)
```

## 3. 취소/환불 플로우

클럽 도메인(club-pseudocode.md)에서 취소를 트리거하면 결제 도메인에 위임된다.

### 3-1. 멤버십 상태별 처리 분기

환불 계산은 `ClubMembership.status = SUBSCRIBED`인 경우에만 적용된다.

```
ClubMembership.status = SUBSCRIBED
  → 환불 금액 계산 후 PG 환불 실행 (3-2 참조)

ClubMembership.status = PAID (ONETIME)
  → 환불 없음
  → ClubMembership.status = EXPIRED, 빌링키 파기

ClubMembership.status = GRACE_PERIOD
  → 환불 없음 (갱신 실패 상태이므로 환불할 최신 결제 없음)
  → ClubMembership.status = EXPIRED, 빌링키 파기

ClubMembership 없음 (무료 클럽)
  → PG 호출 없이 Order.orderStatus → COMPLETED_CANCEL_PAYMENT
```

### 3-2. 환불 금액 계산 (SUBSCRIBED인 경우)

취소 사유(reason)에 따라 환불 정책이 달라진다.

```
예외: 특정 클럽 (clubId 26193, 1758)
  → 환불 0 + Slack 알림 (수동 처리)

ClubMember.status = REQUEST (거절 포함)
  → 전액 환불 (승인 전 상태이므로 사유 무관)

ClubMember.status = APPROVE:

  [MY_SELF (본인 취소) 또는 KICK_BY_SUSPEND (이용정지 강퇴)]
    주문일 기준 14일 이내 → 전액 환불
    주문일 기준 14일 초과 → 환불 없음

  [KICK_BY_HOST (호스트 강퇴)]
    → 전액 환불 (사유 무관)

  [CLOSE_BY_HOST (호스트 폐강)]
    → 전액 환불

  [CLOSE_BY_SYSTEM (시스템 폐강)]
    → 전액 환불
```

### 3-3. 환불 실행

```
refundPrice > 0인 경우:
  Bootpay.cancelReceipt(receiptId, refundPrice)
  → 성공: Order.orderStatus → COMPLETED_CANCEL_PAYMENT
          OrderHistory 기록
  → 실패: Order.orderStatus → FAILED_CANCEL_PAYMENT
          OrderHistory 기록
          Slack 알림 (수동 처리 채널)

refundPrice = 0인 경우:
  PG 호출 없이 Order.orderStatus → COMPLETED_CANCEL_PAYMENT
  OrderHistory 기록

공통 후처리:
  ClubMembership.status = EXPIRED
  빌링키 파기 (Bootpay 빌링키 해지)
```

## 4. 구독 갱신 플로우

```
[스케줄러: 매일 오전 5시]

대상: ClubMembership.status = SUBSCRIBED AND expireAt < NOW() + 1일

처리:
  Bootpay.requestSubscribeCardPayment(billingKey) 청구
  → 성공:
      ClubMembership.expireAt += 1개월
      ClubMembership.status = SUBSCRIBED 유지
      새 Order 생성 (orderStatus = COMPLETED_PAYMENT)
      Payment 생성
      OrderHistory 기록
      Push 발송 (갱신 성공 알림)
  → 실패:
      ClubMembership.status = GRACE_PERIOD
      ClubMember.status = APPROVE 유지 (강퇴 안 함)
      SMS_066 발송 (유예기간 진입 안내)
```

## 5. 주문 상태와 멤버십 상태 정합성

| ClubMember.status | ClubMembership.status | Order.orderStatus | 비고 |
| --- | --- | --- | --- |
| APPROVE (SUBSCRIPTION) | SUBSCRIBED | COMPLETED_PAYMENT |  |
| APPROVE (ONETIME) | PAID | COMPLETED_PAYMENT |  |
| APPROVE (유예 중) | GRACE_PERIOD | COMPLETED_PAYMENT | 갱신 실패, 강퇴 안 함 |
| REQUEST (유료) | SUBSCRIBED / PAID | COMPLETED_PAYMENT |  |
| CANCEL / REJECT (SUBSCRIPTION, 환불 성공) | EXPIRED | COMPLETED_CANCEL_PAYMENT |  |
| CANCEL / REJECT (SUBSCRIPTION, 환불 실패) | EXPIRED | FAILED_CANCEL_PAYMENT |  |
| CANCEL / REJECT (SUBSCRIPTION, 환불 없음) | EXPIRED | COMPLETED_CANCEL_PAYMENT | 14일 초과 본인취소 등 |
| CANCEL / REJECT (ONETIME) | EXPIRED | COMPLETED_PAYMENT | 환불 없음, Order 상태 미변경 |
| CANCEL / REJECT (GRACE_PERIOD) | EXPIRED | COMPLETED_PAYMENT | 환불 없음, Order 상태 미변경 |
| APPROVE / REQUEST (무료) | 없음 | COMPLETED_PAYMENT |  |
| CANCEL / REJECT (무료) | 없음 | COMPLETED_CANCEL_PAYMENT |  |

ONETIME, GRACE_PERIOD 취소 시 Order.orderStatus가 COMPLETED_PAYMENT로 남는 것은 현재 코드의 동작이다.
개선 방향 → `improvements.md` 섹션 5 참조

`FAILED_CANCEL_PAYMENT` 상태는 수동 처리 대기 상태다.
스케줄러 재시도(order-common.md 섹션 4-2) 후에도 해소되지 않으면 수동 환불 후 `COMPLETED_CANCEL_PAYMENT`로 전환한다.
