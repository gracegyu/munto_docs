## **프로젝트 목표**

### **비즈니스 목표**

- **사용자 경험 개선**: 토큰 만료로 인한 예기치 않은 로그아웃 및 세션 끊김 현상 방지
- **중요 플로우 보호**: 결제, VOD 재생, 소셜링 신청 등 중요한 과정에서 토큰 만료로 인한 중단 방지
- **앱 안정성 향상**: 인증 관련 오류 감소로 앱 크래시 및 사용자 이탈 방지

### **기술 목표**

- **토큰 관리 통합**: DI 기반 싱글톤 토큰 서비스로 모든 토큰을 단일 인스턴스로 관리
- **라우팅 시스템 통합**: GoRouter 기반 단일 라우팅 시스템 구축 (현재 5가지 방식 혼재)
- **중요 플로우 보호**: 중요 플로우 진입 시 토큰 유효시간 자동 체크 및 사전 갱신

---

## **핵심 문제점**

### **1. 토큰 관리 시스템**

- **분산된 저장소**: ApiToken(FlutterSecureStorage), SendbirdToken(SharedPreferences), KakaoHelper(자체 TokenManager) 등 토큰별 다른 저장 방식
- **병렬 요청 처리 미흡**: 동시 API 호출 시 토큰 갱신 요청 중복 발생, 서버 부하 및 네트워크 리소스 낭비
- **토큰 생명주기 관리 부재**: 토큰 만료 시간 추적 없음, 중요 플로우 진입 시 사전 체크 없음
- **현재 상태**: TokenInterceptor가 주석 처리되어 실제 동작하지 않음

### **2. 라우팅 시스템**

- **5가지 방식 혼재**: Navigator.push, NavigatorHelper 확장 메서드, RouterMixin, GoRoute, Static push 메서드
- **일관성 부족**: 동일한 화면 이동을 여러 방식으로 처리
- **코드 중복**: 각 화면마다 개별 push 메서드 구현
- **토큰 검증 부재**: 중요 플로우 진입 시 토큰 유효성 사전 체크 없음

## **해결 방안**

### **1. 토큰 서비스 구축**

**구현 내용**:

- DI 컨테이너에 싱글톤으로 등록되는 TokenService 인터페이스 및 구현체 개발
- 모든 토큰을 FlutterSecureStorage로 통합 저장
- 메모리 캐싱을 통한 저장소 접근 최소화

**토큰 갱신 직렬화**:

- Completer 패턴과 요청 큐를 활용한 병렬 요청 제어
- 동시 API 호출 시 단일 토큰 갱신 요청만 진행, 나머지는 대기 후 재시도
- 서버 SSO 환경(유저-토큰 일대다 관계)에서도 클라이언트 측 효율성 보장

**토큰 생명주기 관리**:

- JWT 디코딩을 통한 토큰 만료 시간 추적
- 토큰 만료 시간의 80% 경과 시 자동 갱신 (백그라운드)
- 중요 플로우 진입 시 토큰 유효시간 체크 및 부족 시 즉시 갱신

### **2. 라우팅 시스템 통합**

**GoRouter 기반 통합**:

- 기존 5가지 네비게이션 방식을 GoRouter로 통합
- 모든 화면 이동을 GoRouter를 통한 단일 경로로 처리
- 화면별 개별 push 메서드 제거 및 통합 네비게이션 헬퍼 제공

**라우트 가드 및 미들웨어**:

- 인증 가드: 로그인 필요 화면 진입 시 인증 상태 체크
- 토큰 검증 미들웨어: 중요 플로우 진입 시 토큰 유효시간 체크 및 자동 갱신

**중요 플로우 토큰 보장**:

- 결제 플로우: 최소 10분 토큰 유효시간 보장
- VOD 재생 플로우: 최소 30분 토큰 유효시간 보장
- 소셜링 신청 플로우: 최소 5분 토큰 유효시간 보장
- 프로필 수정 플로우: 최소 5분 토큰 유효시간 보장

**동작 흐름**:

```
중요 플로우 진입 시도
   ↓
토큰 유효시간 체크
   ├─ 충분함 → 즉시 화면 진입
   └─ 부족함 → 로딩 표시 → 토큰 갱신 → 성공 시 화면 진입 / 실패 시 로그인 화면

```

### **3. DI 통합**

**의존성 주입 구조**:

- TokenService를 DI 컨테이너에 싱글톤으로 등록
- ApiClient와 AppRouter에 TokenService 의존성 주입
- 기존 ApiToken 클래스와의 호환성 유지 (점진적 마이그레이션)

## **핵심 스펙**

### **TokenService 인터페이스**

- `getAccessToken()`: 현재 액세스 토큰 조회
- `getRefreshToken()`: 현재 리프레시 토큰 조회
- `setTokens(accessToken, refreshToken)`: 토큰 저장
- `refreshTokens()`: 토큰 갱신 (병렬 제어 포함)
- `clearTokens()`: 토큰 삭제
- `isTokenValid()`: 토큰 유효성 체크
- `getTokenExpirationTime()`: 토큰 만료 시간 조회
- `ensureTokenValid(minRequiredMinutes)`: 토큰 유효성 보장 (필요 시 갱신)

### **라우트 가드 동작**

- 화면 진입 전 `redirect` 콜백에서 인증 상태 및 토큰 유효시간 체크
- 중요 플로우의 경우 최소 필요 시간 보다 토큰 유효시간 보장
- 토큰 갱신 실패 시 로그인 화면으로 리다이렉트

### **호환성 유지**

- 기존 화면 이동 로직 100% 유지
- 기존 API 호출 방식과의 호환성 보장
- 점진적 마이그레이션을 통한 안정성 확보
