# 금칙어 관련 개발 요청

담당부서: 프로덕트
진행상황: 진행중
작업상황: 준비 중
우선순위: 높음

**배경**
플랫폼 내 DM을 통해 외부 채널 유도 및 브로커 메시지 전송 사례가 확인되고 있습니다. 금칙어를 추가 설정하고 탐지를 즉시 확인할 수 있는 체계를 구축하여 브로커 메시지 발송 자체를 줄이는 것을 목적으로 합니다.

---

**우선순위**

| 순위 | 작업 내용 |
| --- | --- |
| 1순위 | 금칙어 다이얼로그 메시지 수정 |
| 2순위 | 금칙어 탐지 시 슬랙 자동 알림 |
| 3순위 | 채팅 데이터 적재 |

1순위와 2순위 병행 가능하다면 함께 진행 부탁드립니다. 순차 진행 시 1순위 완료 후 바로 공유 부탁드립니다. (Sendbird 금칙어 설정을 1순위 완료 후 바로 진행할 예정입니다.)

**5. 채팅 데이터 적재**

- 채팅 메시지 DB 적재
- 이슈 발생 시 개발팀 데이터 요청 방식으로 조회
- 가능할 경우 Grafana / Metabase 조회 및 다운로드

- 이전에 작성한 내용
    
    ## 배경
    
    플랫폼 내 DM을 통해 외부 채널(카카오톡 오픈채팅 등)로 이용자를 유도하거나 브로커 형태의 메시지를 전송하는 사례가 확인
    
    현재는 해당 메시지를 **수동으로 확인 및 대응**하고 있어 탐지와 대응에 한계가 있는 상황. 이에 따라 다음과 같은 기능을 도입하여 **금칙어 기반 메시지 탐지 및 자동 신고 처리 체계**를 구축하고자 함.
    
    - 브로커의 금칙어 메시지 탐지 자동화
    - 금칙어 발송 차단
    - 신고 자동 접수
    
    ---
    
    # 1. 금칙어 탐지 기능
    
    ### 금칙어 목록
    
    ```
    - 무료여성초대
    - 일반모집형 여성 무료
    - party_partners
    - https://open.kakao.com
    ```
    
    **금칙어 변형 패턴**
    
    브로커 메시지는 금칙어를 그대로 사용하기보다 공백, 특수문자, 문자 분리 등을 통해 우회하는 경우가 많아
    아래와 같은 **변형 패턴도 함께 탐지 必**
    
    - 공백 삽입
    - 특수문자 삽입
    - 붙여쓰기 형태
    - 대소문자 변형
    
    예시
    
    ```
    party partners
    party-partners
    partypartners
    open kakao
    ```
    
    가능하다면 **정규식 기반 탐지 적용 가능 여부도 함께 확인 요청**
    
    ### 요구사항
    
    - 메시지 전송 시 금칙어 포함 여부 검사
    - 금칙어 포함 시 메시지 발송 차단
    
    ---
    
    # 2. 메시지 발송 차단
    
    금칙어가 포함된 메시지 전송 시 다음 동작 수행
    
    - Sendbird **Profanity Filter (Block 모드)** 를 통해 메시지 전송 차단
    - **Domain Filter**를 통해 외부 링크 전송 차단
    
    ---
    
    # 3. 자동 신고 처리 및 다이얼로그 노출
    
    금칙어 메시지 발송 시도 시 아래 동작 수행
    
    - 기존 앱의 자동 신고 로직 유지
    - 사용자에게 차단 안내 다이얼로그 노출
    - 자동으로 백오피스 신고 관리 탭에 신고 접수 생성
    
    ### 적용 금칙어
    
    다음 금칙어 관련해서 아래 이미지와 같은 문구 적용
    
    ```
    무료여성초대
    일반모집형 여성 무료
    party_partners
    ```

    # 4. Slack 자동 알림
    
    자동 신고가 생성되면 **Slack 채널로 자동 알림 발송**(새로운 채널 생성 필요)
    
    ### Slack 채널
    
    ```
    #금칙어_자동신고접수
    ```
    
    ### 알림 내용
    
    Slack 알림에는 아래 정보만 포함되면 됨
    
    - 신고 ID
    - 신고 링크 (가능하다면)
    - 탐지된 금칙어
    
    ### 예시
    
    ```
    [금칙어 자동 신고 접수]
    
    신고 ID : 12345
    탐지 금칙어 : party_partners
    신고 링크 : (백오피스 링크)
    ```
    
    ---
    
    # 5. 금칙어 관리 기능
    
    ### 요구사항
    
    - 금칙어 추가 가능
    - 추가된 금칙어도 동일 로직 적용
    
    즉
    
    금칙어 추가 시 자동으로
    
    - 메시지 발송 차단
    - 자동 신고 생성
    - Slack 알림 발송
    
    ---
    
    # 6. 채팅 데이터 저장
    
    채팅 메시지는 DB에 적재
    
    ### 조회 방식
    
    관리자 검색 기능은 필수 아니며 아래 방식으로 진행
    
    - 특별 이슈 발생 시 개발팀 데이터 요청
    - 가능할 경우, Grafana / Metabase 데이터 조회 및 다운로드
