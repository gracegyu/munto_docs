# UDID 기반 접속 제한 구현

| 항목 | 내용 |
| --- | --- |
| 배경 | 소셜링 부정 이용(한 계정을 다수 기기에서 접속해 자리를 대리 구매하는 브로커 행위 등)이 확인되어, Notion 문서 「UDID 기반 접속 제한 정책」에 따라 디바이스 식별자(UDID)를 활용한 접속 제한을 도입하려고 합니다.<br><br>[참고 문서: UDID 기반 접속 제한 정책 (Notion)](https://www.notion.so/326e2bc7639d8055976df300842c9aca?pvs=21) |
| 현재 문제점 | - 동일 UDID·다계정·다기기에 대한 체계적 제한이 없어 부정 이용에 악용될 수 있는 여지가 있었습니다.<br>- 운영 관점에서 기기 단위·계정 단위의 동시 접속 한도를 명확히 적용하기 어려웠습니다. |
| 개선 방향 | - 단일 기기(UDID) 다중 계정: 동일 UDID에서 로그인 이력이 2개인 상태에서 신규 3번째 계정 로그인·가입 시도 시 차단(해당 기기 로그인 이력 전체를 기간 제한 없이 기준으로 적용).<br>- 단일 계정 다중 기기: 1계정당 최대 2기기까지 허용하고, 3번째 기기 접속 시도부터 차단합니다.<br>- 웹 브라우저 접속은 UDID 수집 대상이 아니므로 본 정책 적용 범위에서 제외합니다.<br>- 차단 발생 시 UDID·시도 일시 등 차단 이력을 DB에 기록합니다. |
| 고려요소 | - 정책·개인정보·이용약관 관점의 검토 및 고지 체계가 필요합니다.<br>- 웹 제외, 기기 변경 등 예외·오탐 최소화를 위한 모니터링과 단계적 조정이 필요할 수 있습니다.<br>- 문서상 체험단 런칭 후 UDID 활동 데이터 모니터링 결과에 따라 조치를 조정할 수 있습니다. |
| 기대효과 | 브로커·부정 예약 완화, 계정·기기 단위 정책의 일관된 적용, 운영 대응 근거(로그·차단 이력) 확보가 기대됩니다. |
