# [클럽] 무료 승인제 클럽 24시간 자동취소 시 Order 상태 불일치 데이터 픽스

## **배경**

APPF-803 원인 분석 과정에서 발견된 기존 데이터 불일치.

`[WEBB-1005]` (2026-03-03)에서 무료 클럽도 V3 Order를 생성하도록 변경했으나, 스케줄러 24시간 자동취소 경로에서 Order 취소 처리가 누락됨.

`[WEBB-1155]` (2026-03-18)에서 일부 경로(호스트 거절, 유저 취소)는 수정됐으나 스케줄러 경로는 수정되지 않아 2026-03-03 ~ 2026-03-18 사이에 24시간 자동취소된 무료 클럽 신청 건의 Order가 `COMPLETED_PAYMENT`로 잔존.

## **영향**

해당 유저들이 동일 클럽에 재신청 시 `POST /api/v3/order/club/{id}` 에서 **409 Conflict** 반환 → 신청 불가

## **대상 데이터 규모**

Prod DB 기준 **990건**

- ClubMember `CANCEL` (24시간 자동취소) + Order `COMPLETED_PAYMENT` 불일치: **842건**
- ClubMember `REJECT` (호스트 거절) + Order `COMPLETED_PAYMENT` 불일치: **148건**

> REJECT 148건은 스케줄러가 아닌 다른 경로로 발생했을 수 있으므로 별도 검토 필요
> 

## **조회 쿼리**

```
SELECT COUNT(*)
FROM "Order" o
JOIN "ClubMember" cm ON cm."userId" = o."userId" AND cm."clubId" = o."clubId"
WHERE o."orderStatus" = 'COMPLETED_PAYMENT'
  AND o."cancelledAt" IS NULL
  AND o."clubId" IS NOT NULL
  AND o."clubMembershipId" IS NULL
  AND cm.status IN ('CANCEL', 'REJECT')
  AND (cm.status != 'CANCEL' OR cm."canceledAt" IS NOT NULL);
```

## **수정 방법**

아래 SQL을 마이그레이션 스크립트로 실행 (`OrderHistory` 이력 포함 처리 필요)

```
SELECT COUNT(*)
FROM "Order" o
JOIN "ClubMember" cm ON cm."userId" = o."userId" AND cm."clubId" = o."clubId"
WHERE o."orderStatus" = 'COMPLETED_PAYMENT'
  AND o."cancelledAt" IS NULL
  AND o."clubId" IS NOT NULL
  AND o."clubMembershipId" IS NULL
  AND cm.status IN ('CANCEL', 'REJECT')
  AND (cm.status != 'CANCEL' OR cm."canceledAt" IS NOT NULL);
```

> `OrderHistory` 이력도 함께 INSERT 필요
>
