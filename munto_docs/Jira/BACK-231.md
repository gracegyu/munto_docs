# [백오피스] v1→v2 마이그레이션 및 UI 전체 업데이트 — PoC 검증 후 OnePager 작성

| 항목 | 내용 |
| --- | --- |
| 배경 | 현재 백오피스는 v1·v2가 공존하는 상태로, 기존 기능은 v1에서, 최근 개발된 기능은 v2에서 제공되고 있다. 이로 인해 운영자가 두 버전을 번갈아 사용해야 하는 불편이 발생하고 있다. BACK-221 검토 결과를 바탕으로, v1 기능 전체를 v2로 마이그레이션하고 UI를 통합하는 작업의 실현 가능성을 PoC로 검증하고자 한다. |
| 현재 문제점 | ① 기존 기능(v1)과 신규 기능(v2)이 분리된 채 운영되어 사용자 경험이 분산된다. ② 신규 화면 추가 시마다 기획-개발 간 반복 커뮤니케이션이 발생한다. |
| 개선 방향 | v1 기능을 v2로 마이그레이션하면서 UI 전체를 기획자 제공 디자인 화면 기반으로 디벨롭하는 PoC를 수행한다. PoC 완료 후 **1) v1→v2 마이그레이션 범위·방식**, **2) 구현 가능성**, **3) 추가 명세 필요 항목**을 검증하고 결과를 OnePager로 정리한다. OnePager에는 작업 범위, 진행 방식, 예상 일정, 리스크를 포함한다. |
| 고려요소 | 기획자-개발자 협업 워크플로: 디자인 화면 전달 형식과 Claude 입력 방식 간 표준을 정립해야 한다. |
| 기대효과 | 기획자가 디자인 의도를 직접 화면에 반영하는 워크플로가 정립되어 기획-개발 간 반복 수정 횟수가 줄어든다. PoC 결과를 바탕으로 작성된 OnePager가 백오피스 v2 UI 전체 업데이트 본 작업의 의사결정 근거로 활용된다. |
