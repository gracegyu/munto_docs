# 문토 보안 개발 가이드

## 1. 목적 및 배경

* 문토는 **소셜 서비스 플랫폼**으로, 다수의 고객 개인정보(이메일, 전화번호, 프로필 사진, 대화 내용 등)를 처리합니다.
* 최근 개인정보 유출에 따른 법적·사회적 책임이 강화되고 있으며, 보안 위반은 기업 신뢰도에 치명적 손실을 초래합니다.
* 본 가이드는 개발자가 소스코드 작성 단계부터 배포까지 전 과정에서 **개인정보 보호**와 **보안 강화**를 실천하도록 지원하기 위해 수립되었습니다.

---

## 2. 적용 범위

* 모든 개발자 및 협력사 엔지니어
* 문토의 **백엔드, 프론트엔드, 모바일 앱, 데이터베이스, 운영 인프라** 전체
* 신규 프로젝트 및 기존 코드 유지보수 모두 적용

---

## 3. 개인정보 정의

개인정보는 사용자를 특정할 수 있는 모든 정보를 포함합니다. (예시)

* 기본 정보: 이름, 전화번호, 이메일, 생년월일
* 민감 정보: 위치 정보, 건강 정보, 금융 정보
* 서비스 내 정보: 프로필 이미지, 대화 내용, 소셜 활동 기록

---

## 4. 개발 시 개인정보 취급 원칙

### (1) 최소 수집·최소 사용 원칙

* 불필요한 개인정보를 수집하지 않는다.
* 수집 목적 외의 용도로 사용하지 않는다.
* 로그, 임시 데이터, 테스트 데이터에 개인정보를 남기지 않는다.

### (2) 코드 레벨 보안 규칙

* **파일명/URL에 개인정보 포함 금지** (예: 전화번호, 이메일 → 금지)
* **주석/로그에 개인정보 기록 금지**
* **DB 스키마 설계 시 암호화·익명화 고려**
* **환경변수(.env)로 민감정보 관리**, 코드에 하드코딩 금지

### (3) 데이터 저장·전송 보안

* 비밀번호는 단방향 해시(예: bcrypt, Argon2)로 저장
* 개인정보 저장 시 AES256 등 강력한 암호화 적용
* 모든 API 통신은 HTTPS/TLS 1.2 이상 강제

### (4) 접근 제어

* 개인정보 접근 함수/쿼리는 역할(Role)에 따라 제한
* 관리자 콘솔 접근은 MFA(다중 인증) 적용
* 접근 로그를 반드시 남기고, 이상 접근 탐지

---

## 5. 리뷰 및 점검 프로세스

### (1) 코드 리뷰 체크리스트 (예시)

* [ ] 개인정보가 URL, 파일명, 로그, 주석에 노출되지 않았는가?
* [ ] 불필요한 개인정보를 DB에 저장하지 않았는가?
* [ ] 암호화/해시가 필요한 항목은 적절히 처리되었는가?
* [ ] 외부 API 호출 시 개인정보 전송이 안전한가?
* [ ] 인증/인가 로직이 누락되지 않았는가?

### (2) 정기 보안 점검

* 분기별 **개발 보안 점검 회의** 진행
* 보안 담당자와 함께 **소스 코드 보안 스캐닝(SonarQube, Snyk 등)**
* **취약점 리포트**를 Jira로 관리 및 추적

---

## 6. 재발 방지 체계

* 신규 입사 개발자 교육 시 **보안 개발 가이드** 필수 학습
* 사고 발생 시 **RCA(Root Cause Analysis)** + 가이드 업데이트
* 보안 전담자 지정 → 개선 조치 추적

---

## 7. 문토 특화 강화 조치

* **소셜 서비스 특성상 사진/미디어 업로드가 많음**

  * 업로드 시 파일명 자동 변환(UUID/UnixTime 기반)
  * CloudFront/S3 접근 제어 정책 적용 (직접 접근 차단, Presigned URL 사용 검토)
* **외부 공유 기능(Facebook, Instagram 등)**

  * 외부 API 정책 변경 시 즉각 점검
  * 개인정보가 외부로 전송되지 않도록 필터링

---

## 8. 결론

* 본 가이드는 단순 권고가 아니라 **필수 준수 사항**입니다.
* 모든 개발자는 보안 관점에서 코드를 작성해야 하며, 이를 어길 경우 서비스와 고객에 심각한 피해가 발생할 수 있습니다.
* 문토의 보안 문화는 **"개인정보는 절대 노출하지 않는다"**는 원칙에서 시작합니다.
