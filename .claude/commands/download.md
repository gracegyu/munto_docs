# Notion 문서 다운로드

Notion 문서를 다운로드하여 로컬 마크다운 파일로 저장합니다.

**중요: Notion URL 또는 페이지 ID로 문서를 가져옵니다.**

---

## 단계 1: Notion URL 또는 페이지 ID 받기

사용자에게 질문:

```
어떤 Notion 문서를 다운로드하시겠습니까?
(Notion 페이지 URL 또는 페이지 ID를 입력하세요)

예시:
- https://www.notion.so/munto/Page-Title-abc123...
- abc123def456... (페이지 ID만)
```

URL이나 ID를 받으면 **즉시 단계 2로 이동**.

---

## 단계 2: Notion 페이지 내용 가져오기

Notion MCP 도구를 사용하여 페이지 내용을 가져옵니다.

**사용할 도구:**

```
mcp_Notion_notion-fetch

parameters:
{
  "id": "[페이지 URL 또는 ID]"
}
```

페이지 내용을 가져온 후 **단계 3으로 이동**.

---

## 단계 3: 저장 경로 결정

사용자에게 질문:

```
다운로드한 문서를 어디에 저장하시겠습니까?

1. 기본 경로 사용: my_docs_dev/[문서제목].md
2. 직접 경로 입력

선택: (엔터 = 기본 경로)
```

**기본 경로 사용 시:**

- 페이지 제목을 파일명으로 사용
- `my_docs_dev/` 디렉토리에 저장
- 특수문자는 제거하고 공백은 언더스코어로 변환
- 예: "AI 기반 추천 시스템" → `my_docs_dev/AI_기반_추천_시스템.md`

**직접 입력 시:**

- 사용자가 지정한 경로 사용
- 디렉토리가 없으면 생성 안내

---

## 단계 4: 파일 중복 확인 (필수)

파일을 저장하기 전에 동일한 경로에 파일이 이미 존재하는지 확인합니다.

**확인 방법:**

- `read_file` 도구로 파일 읽기 시도
- 파일이 존재하면 오류가 발생하지 않음
- 파일이 없으면 "Could not find file" 오류 발생

**파일이 이미 존재하는 경우:**

사용자에게 질문:

```
⚠️ 파일이 이미 존재합니다: [파일 경로]

어떻게 하시겠습니까?
1. 덮어쓰기 (Overwrite) - 기존 파일을 삭제하고 새로 저장
2. 취소 (Cancel) - 다운로드를 중단
3. 다른 이름으로 저장 - 새 파일명 입력

선택 (1/2/3):
```

**사용자 선택에 따른 동작:**

1. **덮어쓰기 선택 시:**

   - 기존 파일을 그대로 덮어씀
   - 단계 5로 이동

2. **취소 선택 시:**

   - 다운로드 중단
   - "다운로드가 취소되었습니다." 메시지 표시
   - 종료

3. **다른 이름으로 저장 선택 시:**
   - 새 파일명 입력받기
   - 제안: `[원본파일명]_[날짜].md` 또는 `[원본파일명]_2.md`
   - 새 경로로 단계 5 진행

**파일이 존재하지 않는 경우:**

- 바로 단계 5로 이동

---

## 단계 5: 마크다운 파일로 저장

가져온 Notion 페이지 내용을 마크다운 형식으로 변환하여 저장합니다.

**저장 형식:**

Notion에서 가져온 내용을 그대로 저장합니다.

- 메타데이터(출처, 날짜 등)를 추가하지 마세요
- Notion 내용 그대로 마크다운 파일로 저장

```markdown
[Notion 페이지 내용 그대로]
```

**사용할 도구:**

- `write` 도구로 파일 저장

---

## 단계 6: 결과 보고

성공:

```
✅ Notion 문서를 다운로드했습니다!

제목: [문서 제목]
저장 경로: [파일 경로]
파일 크기: [줄 수] 줄

원본 Notion URL: [URL]
```

실패:

```
❌ 다운로드에 실패했습니다.
오류: [오류 메시지]

해결 방법:
- Notion 페이지 URL이 올바른지 확인
- Integration이 해당 페이지에 접근 권한이 있는지 확인
- 페이지가 삭제되지 않았는지 확인
```

## 추가 기능

### 여러 페이지 다운로드

사용자가 여러 URL을 입력한 경우:

```
다음 문서들을 다운로드하시겠습니까?
1. [제목1] - [URL1]
2. [제목2] - [URL2]
3. [제목3] - [URL3]

모두 다운로드? (y/n)
```

### 이미지 처리

Notion 페이지에 이미지가 있는 경우:

```
⚠️ 이 문서에 [N]개의 이미지가 포함되어 있습니다.

이미지는 Notion 링크로 참조됩니다.
별도로 다운로드하려면 수동으로 처리가 필요합니다.
```

### 하위 페이지 처리

Notion 페이지에 하위 페이지가 있는 경우:

```
⚠️ 이 페이지에 [N]개의 하위 페이지가 있습니다.

하위 페이지도 함께 다운로드하시겠습니까? (y/n)
```

---

## 주의사항

1. **Notion Integration 권한**

   - 다운로드하려는 페이지가 Integration과 공유되어 있어야 함
   - Notion에서 페이지 우측 상단 `...` → `Connections` 확인

2. **마크다운 변환 제한**

   - 일부 Notion 블록(데이터베이스 뷰, 임베드 등)은 완벽히 변환되지 않을 수 있음
   - 복잡한 레이아웃은 단순화될 수 있음

3. **이미지 및 파일**

   - 이미지는 Notion URL로 참조됨
   - 파일 첨부는 링크로 표시됨
   - 로컬 저장을 원하면 수동 다운로드 필요

4. **Database 페이지**
   - Database 자체는 다운로드 불가
   - Database 내의 개별 페이지는 다운로드 가능

## 예시 사용

```
사용자: claude download

AI: 어떤 Notion 문서를 다운로드하시겠습니까?
사용자: https://www.notion.so/munto/AI-abc123...

AI: [Notion 페이지 가져오기]
문서 제목: "AI 기반 추천 시스템 설계서"

저장 경로: (엔터 = my_docs_dev/AI_기반_추천_시스템_설계서.md)
사용자: [엔터]

AI: [파일 저장]
✅ Notion 문서를 다운로드했습니다!

제목: AI 기반 추천 시스템 설계서
저장 경로: my_docs_dev/AI_기반_추천_시스템_설계서.md
파일 크기: 1250 줄
```

## 고급 옵션

### 여러 문서 일괄 다운로드

```
Database의 모든 페이지를 다운로드하려면:
1. Database URL 제공
2. Database 내 모든 페이지 URL 목록 가져오기
3. 각 페이지를 개별 파일로 저장
```

### 폴더 구조 유지

```
Notion의 계층 구조를 로컬 폴더 구조로 변환:
- 최상위 페이지: my_docs_dev/[제목].md
- 하위 페이지: my_docs_dev/[제목]/[하위제목].md
```
