# 이슈 생성 커맨드

사용자가 제공한 이슈 제목으로 APPF 프로젝트에 개선 작업 이슈를 생성합니다.

## 작업 순서

1. 먼저 APPF 프로젝트의 이슈 생성 메타데이터를 조회합니다
2. 사용 가능한 이슈 타입 ID와 필수 필드를 확인합니다
3. 다음 5개 항목이 포함된 표 형태의 이슈를 생성합니다:

```
| 항목 | 내용 |
|------|------|
| 배경 | 개선이 필요한 이유 |
| 현재 문제점 | 개선 전 문제나 불편 사항 |
| 개선 방향 | 개선하고자 하는 구체적 방향 |
| 고려요소 | 성능, 보안, UX 등 개선 시 주의사항 |
| 기대효과 | 개선 완료 후 예상되는 효과 |
```

4. 이슈 생성 후 이슈 번호와 URL을 사용자에게 알려줍니다

## 프로젝트 정보
- 프로젝트 키: APPF
- 프로젝트 ID: 10018
- 이슈 타입: 개선 (Improvement) 또는 Task

## 주의사항
- 이슈 제목은 명확하고 구체적으로 작성
- 각 항목은 마크다운 표 형태로 작성
- 생성 후 반드시 이슈 URL을 반환
